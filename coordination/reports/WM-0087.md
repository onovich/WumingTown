# WM-0087 Work Report

## Outcome

Added a reproducible Chrome Stable / Edge Stable Web gate measurement script,
ran built-artifact loading and shell interaction evidence in both browsers, and
stabilized the benchmark gate after review exposed both `entity-store`
outlier sensitivity and `map-dirty` microbenchmark timer noise on the current
Windows / Node environment.
The current Web path still only supports a `demo-only` verdict at best.
Loading and compressed download evidence are strong, JS heap samples show no
monotonic growth on the current shell, and the final required full-suite bench
gate now passes from a temp artifact root without changing baselines or
thresholds. Same-spec remains blocked because the measured browser path is
still a read-only fixture consumer and the current M5 browser Worker path is
checkpoint-projection evidence rather than a real 30 TPS / 20k-entity browser
authority runtime.

## Decisions and deviations

- Used a temp-path JSON artifact for browser evidence so the task does not
  rewrite tracked benchmark or screenshot artifacts.
- Initial raw `pnpm bench` on the default WM-0083 artifact path produced a
  one-run `m5-alpha-content-long-run` outlier and dirty tracked benchmark
  outputs; those tracked outputs were restored immediately.
- Second-round review then reproduced a different blocker: default full-suite
  sampling could let `entity-store` bimodal slow samples dominate the median on
  this Windows / Node environment.
- WM-0087 therefore changed only the default benchmark sampling window from
  `5/1` to `9/2`, matching the focused reviewer-guided diagnosis where
  `pnpm bench --filter entity-store --samples 9 --warmup 2` and
  `pnpm bench --samples 9 --warmup 2` were stable and passing. Thresholds,
  baselines and benchmark invariants remained unchanged.
- Third-round review then identified a separate reviewer-risk on `map-dirty`:
  a sub-`1 ms` baseline leaves only `0.2 ms` before the blocking threshold, so
  single-pass timing on this Windows / Node environment is too sensitive to
  scheduler noise.
- The final `map-dirty` fix is intentionally conservative. It keeps the
  original `dirtyCellsInsideChunk(...)`, coordinate-loop, inline update object,
  `grid.updateCell(x, y, {...})` path and original timer window. WM-0087 adds
  only `16` repeated identical passes per sample and reports the average
  elapsed time while reusing the final pass's deterministic invariants.
- Recorded JS heap stability through forced-GC CDP samples because total browser
  memory APIs were not reliably available from the current Chromium setup;
  `measureUserAgentSpecificMemory` was unavailable in both browsers.
- Treated `interaction latency P95` as the current main-thread evidence for the
  shell, not as proof of a full product-scale tick budget.
- Explicitly reported the product-scale 30 TPS gate as blocked because:
  - the default Web shell mounts a read-only fixture; and
  - `packages/sim-worker/src/index.ts` uses `createM5WorkerProjection(...)` for
    the M5 browser Worker path, while
    `packages/sim-core/src/m5-worker-projection.ts` serves fixed checkpoint data
    with `entityCount: 7` rather than a measured continuous browser runtime.
