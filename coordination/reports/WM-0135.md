# WM-0135 Work report

## Outcome

Implemented the post-M8 responsive layout gate for the player Web shell. The HUD now exposes explicit `desktop`, `medium` and `compact` layout modes, with required windowed sizes using a medium composition that keeps a measurable Pixi map focus area visible while moving long labels and secondary cards into stable scroll regions. The change is presentation-only and does not change simulation authority, save format, release verdicts or platform status.

## Files and public contracts changed

- `packages/ui-react/src/shell-hud.ts`: adds responsive HUD mode selection, a medium player HUD layout, medium debug overlay constraints, and stable map-focus / bottom-drawer test slots.
- `packages/ui-react/src/shell-hud.test.ts`: covers layout-mode thresholds for desktop, medium and compact player HUD render paths.
- `apps/web/src/web-shell.e2e.test.ts`: strengthens the responsive matrix to assert HUD layout mode, map-focus geometry, no incoherent HUD coverage and reviewer-readable DOM layout artifacts.
- `coordination/reports/WM-0135-plan.md`: records the scoped implementation plan.
- `coordination/reports/WM-0135-responsive-layout.md`: stores the DOM-layout evidence generated by the responsive E2E matrix.
- `coordination/tasks/WM-0135.json`: task state/history only, from claim/complete workflow.

## Acceptance evidence

- Covers required viewports: `coordination/reports/WM-0135-responsive-layout.md` records 1280x720, 1366x768, 1424x861, 1600x900, 1920x1080, 2560x1369 and 2560x1440.
- Runs windowed/fullscreen-equivalent checks in zh-CN and en: the artifact records both locales for start surface and player HUD at every required viewport, plus debug overlay evidence.
- Map remains inspectable and panels do not incoherently cover the play surface: E2E now asserts `player-map-focus` size, viewport bounds, canvas coverage and lack of HUD coverage; the artifact records map focus dimensions for every player HUD viewport.
- Long labels wrap or move into stable scroll regions without overlap: medium mode moves dense secondary cards into `player-bottom-drawer`, and E2E uses clipped ancestor-aware reachability checks for long scrollable panels.
- Screenshot or DOM-layout artifacts stored: DOM artifact is stored at `coordination/reports/WM-0135-responsive-layout.md`.
