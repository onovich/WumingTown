# WM-0147 Work Report

## Outcome

Reworked the default player HUD so the first screen reads like a playable game
surface instead of a diagnostics/status board. The map remains the dominant
visual subject, the default HUD now emphasizes one objective plus one obvious
action path, and settings/locale/system preferences move behind an explicit
secondary surface.

This task stays inside client/UI scope. It does not change Simulation Worker
protocol, sim-core runtime behavior, authoritative world ownership, release
verdicts, public saves, telemetry, accounts, or store/public-distribution
surfaces.

## Files and public contracts changed

- `packages/ui-react/src/shell-hud.ts`: rebuilt the default HUD information
  hierarchy around objective/action flow, moved settings behind an explicit
  toggle surface, kept the bottom command bar plus selected-object inspector
  readable across layout modes, and kept the explicit diagnostics overlay from
  covering the player command bar.
- `packages/ui-react/src/shell-main-menu-surface.ts`: simplified the start
  surface so first-play guidance stays concise and settings/language controls
  are hidden behind the explicit settings view.
- `packages/ui-react/src/shell-hud.test.ts`: updated unit coverage for the new
  objective/action hierarchy and the hidden-by-default settings surface.
- `apps/web/src/web-shell.e2e.test.ts`: updated responsive/E2E assertions for
  the simplified HUD, added explicit settings-surface checks, and made
  WM-0140 screenshot capture opt-in via `WM0140_CAPTURE_ARTIFACTS=1` so
  routine `web-shell` verification does not persist out-of-scope
  `tools/artifacts/**` changes during WM-0147.
- `coordination/reports/WM-0147-plan.md`: execution plan.
- `coordination/reports/WM-0147.md`: this report.
- `coordination/tasks/WM-0147.json`: task claim metadata only.

## Acceptance evidence

- Concise objective/action hierarchy:
  the default HUD removes the always-expanded current-state, first-play
  guidance, task list, event list and settings stack from the first screen.
  The primary path is now `next goal` + `attention queue` + `command bar`.
- Map-first first impression:
  the world canvas remains full-bleed behind the HUD and the top-level layout
  no longer competes with diagnostics/status-board copy for first-screen
  attention.
- Secondary information hidden behind explicit surfaces:
  locale, UI scale and related display/system preferences now stay behind the
  main-menu settings view and the in-HUD settings toggle surface instead of
  being always visible.
- Command clarity with disabled reasons:
  the bottom command bar still exposes "what can I do now", while disabled
  commands continue to provide `aria-describedby` detail text for unavailable
  actions.
- Responsive and bilingual coverage:
  the updated unit/E2E assertions cover `en` and `zh-CN`, and the viewport
  checks explicitly verify 1280x720, 1366x768, 1600x900 and 1920x1080 in the
  responsive matrix without reintroducing forbidden first-screen settings or
  diagnostics surfaces.

## Visual evidence approach

WM-0147 does not persist reviewer screenshots into `tools/artifacts/**`
because the task packet does not allow that path. Verification evidence for
this task is therefore based on DOM assertions, viewport-bound checks, locale
checks and command-state assertions in `shell-hud.test.ts` and
`web-shell.e2e.test.ts`. WM-0140 screenshot capture remains available only as
an explicit opt-in path via `WM0140_CAPTURE_ARTIFACTS=1`.
