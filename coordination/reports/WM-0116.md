# WM-0116 Work report

## Outcome
Default Web and desktop launch now open on a localized player-facing main menu instead of dropping directly into the shell HUD or diagnostics harness. The new surface provides `New Game`, `Continue`, `Settings`, and language controls, reuses the existing locale-preference infrastructure, and now stays viewport-bounded at compact sizes by top-anchoring the panel and allowing polished internal scrolling so the language affordances remain reachable on short/mobile viewports.

## Files and public contracts changed
- `packages/ui-react/src/shell-main-menu-surface.ts`: new default start/settings surface with localized actions, quick language switching, and a settings view that reuses the existing shell locale panel.
- `packages/ui-react/src/shell-hud.ts`: show the main menu by default on player launch, keep diagnostics-only launch paths unchanged, and reveal the existing HUD after `New Game` / `Continue`.
- `packages/ui-react/src/localization.ts`: added `ui.mainMenu.*` keys for zh-CN/en menu, settings, and language-switch copy.
- `packages/ui-react/src/shell-hud.test.ts`: updated render coverage for the new default menu surface in English and zh-CN.
- `apps/web/src/web-shell.e2e.test.ts`: assert default main-menu launch, settings entry, language switching, persistence after reload, and HUD behavior after dismissing the start surface.
- `apps/desktop-electron/src/desktop-shell.e2e.test.ts`: assert the same main-menu flow for default desktop launch while preserving explicit diagnostics-mode expectations.
- `coordination/tasks/WM-0116.json`: task claim metadata and `in_progress` state for this work.

  `corepack pnpm test:e2e --filter web-shell` and `corepack pnpm test:e2e --filter desktop-shell` both passed.

## Determinism, save, performance and security impact
Determinism: none. No simulation or Worker logic changed.

Save: no schema or persistence-surface expansion. `Continue` only reuses the existing shell storage load action; locale persistence still uses the existing app-shell preference storage.

Performance: low-risk UI-only change. The main menu is a bounded React surface rendered once per launch path; no new sim hot-path work or per-tick allocations were added in Worker code.

Security: none. No Electron preload API, sandbox setting, IPC surface, signing, telemetry, or account flow changed.

## Decisions and deviations
- Reused the existing WM-0114/WM-0115 locale infrastructure and `ShellSettingsPanel` instead of introducing a second settings or i18n path.
- Kept explicit diagnostics launch (`wmDiagnostics=1`) on the existing debug-oriented shell flow so reviewer and dev diagnostics coverage remain intact while default player launch becomes the main menu.
- Did not use Spark/rapid-implementer. The task relied on screenshot-driven UI interpretation and coordinated Web plus desktop E2E updates, so it stayed with the `client-engineer` owner role.
- Relevant contracts: `docs/05_tech/12_m8_i18n_locale_contract.md`, `docs/01_design/09_m8_product_ui_design_system.md`, `coordination/decisions/ADR-0011.md`.
