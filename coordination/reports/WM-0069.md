## Determinism, save, performance and security impact
Determinism: Worker command ids use the WM-0068 M4 command-id parser and match
the M4 replay checkpoint sequence. Divergence diagnostics name the first
mismatched checkpoint tick.

Save/protocol: the Worker reuses existing `RenderSnapshot`, `UiDelta`,
`MetricsSample` and `SaveReady` message kinds. M4 save metadata is limited to
scenario id, checkpoint tick and world hash, matching existing focused-save
behavior.

Performance: diagnostics record snapshot/read-model/projection byte sizes and
message counts. M4 Worker projections read the focused scenario read model and
rebuilt surface hashes; they do not become persisted cache authority.

Security and authority: UI and browser Worker consumers receive read-only
projections only. No UI, Electron, React, Pixi or client code can mutate M4
lamp, evidence, obligation, town-rule, crisis or director facts through this
task.

## Decisions and deviations
`packages/sim-worker/src/index.ts` was already over the 400-line preference.
This task adds the M4 branch to the existing central protocol dispatcher rather
than splitting the Worker protocol surface during a parity task. A later
cleanup can extract per-scenario projection adapters if the Worker gains more
scenario branches, but doing that here would have broadened the protocol
change.

## Remaining risks / follow-up tasks
- WM-0069 does not add benchmark baselines or threshold changes; that remains
  WM-0070 scope.
- M4 projections are intentionally narrow and read-only. Public protocol
  versioning or richer structured projection payloads require a separate
  architecture gate.
