## Acceptance evidence

- Random calls require an explicit named stream: `createNamedRandomStreams().nextUint32("combat")` and `.nextUint32("social")` are covered by `packages/sim-core/src/determinism.test.ts`; empty stream names throw.
- RNG state survives serialize/restore: `serializeHeadlessRunner` and `restoreHeadlessRunner` are covered by the round-trip determinism test, then both original and restored runners advance to identical summaries.
- RNG restore validation rejects inconsistent stream ownership and invalid xorshift state: the regression test mutates the nested RNG snapshot seed and a restored stream state of `0`; both are rejected.
- Canonical world hash ignores presentation state and order accidents: `formatCanonicalWorldHash` test uses reordered fields, streams, commands, and different presentation payloads with identical hash output.
- Canonical world hash ignores scheduler/playback controls: reviewer-response regression mutates only `speed` and `paused` after two runners reach the same authoritative tick/history; `worldHash` remains equal while summary control fields differ.
- Canonical world hash rejects duplicate names: the regression test rejects duplicate canonical field names and duplicate random stream names before hashing.
- Perturbed replay identifies first divergent checkpoint: unit test and `pnpm sim:replay-test` both perturb checkpoint index `2` at tick `6` and verify `world_hash_mismatch`.

## Determinism, save, performance and security impact

Determinism: improved with explicit named streams, canonical hash input ordering, duplicate-name rejection, scheduler/playback control exclusion, and replay divergence reporting. Save/replay: headless snapshots now include RNG stream state, reject nested RNG seed mismatches, reject zero restored stream state, and restore valid RNG state before further ticks. Performance: hashing/snapshot sorting uses bounded manual insertion for non-hot checkpoint/summary paths; per-tick RNG is a small integer xorshift update. Security/runtime boundaries: `sim-core` did not add DOM, React, Pixi, Electron, Node filesystem, real time, ambient randomness, async, or new runtime dependencies.

## Decisions and deviations

No ADR required; no new dependency or public worker protocol was added. No Spark used per the hard boundary. The implementation intentionally keeps this as a foundation rather than a full save-file schema or cross-version migration layer.

Reviewer response: removed `paused` and `speed` from canonical world hash input because they are scheduler/playback controls, not authoritative world state. Added regression coverage proving equal authoritative state/tick/command history keeps the same `worldHash` even when summary `speed` and `paused` differ.

## Remaining risks / follow-up tasks

- Snapshot restore now validates internal seed consistency and stream state invariants, but still accepts typed in-process snapshot objects; future persistence work should add unknown-data schema validation before loading external save files.
- Canonical hashing currently covers the existing headless authoritative state. Future component stores should add explicit canonical serializers as new authoritative state is introduced.
- Replay diagnostics compare prepared checkpoints; future replay tooling can attach command-log minimization and richer report artifacts.
