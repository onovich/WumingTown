## Implementation summary

- Added structured shortage reasons:
  `reservation_insufficient_amount` and
  `reservation_insufficient_capacity`.
- Kept conflict reasons for true active-claim contention:
  `reservation_item_quantity_conflict` and
  `reservation_capacity_conflict`.
- Hardened `ReservationLedger.restoreFromSnapshot` so record-count,
  active-count and ledger-version shape mismatches fail before clearing the
  current ledger.
- Mapped the new shortage reasons at build-site reservation call sites without
  expanding build/order behavior.
- Added focused tests for M2 multi-channel atomicity, stale target rejection,
  invalid owner rejection, invalid parameter rejection, terminal complete/fail/
  cancel cleanup, destroy cleanup through existing lifecycle hooks, and
  load-rebuild clearing/rebuild.
- Added an exported focused M2 reservation contention benchmark helper. The CLI
  benchmark filter registry is outside WM-0037 allowed paths, so it was not
  modified.
