# WM-0054 Work report

## Outcome

Implemented the focused M3 mood, thought, and memory owner surface in
`sim-core`: fact-driven mood targets from needs, environment, and health facts;
bounded deterministic thought and memory retention; scheduled integer current
mood convergence; structured mood reason traces; metrics and replay-stable
hash evidence. No UI, Worker protocol, save/schema, package, dependency,
relationship graph, M4, or real-time/random authority changes were made.

Changes-requested repair: cross-actor row eviction now updates the actor whose
row was removed. Evicted thought rows immediately recalculate that actor's mood
target and advance its owner version; evicted memory rows advance the affected
actor version because memories are owner facts but do not feed mood target
calculation.

## Determinism, save, performance and security impact

Determinism: all rows use stable actor/row ordering, fixed integer lanes,
bounded capacities, explicit stack keys, source versions, and deterministic
hashing. No `Math.random`, real time, platform clocks, DOM, React, Pixi,
Electron, Node filesystem, Promise/coroutine execution state, or ambient
entropy is used.

Save/replay: no public save format or schema changed. `createHash()` provides
focused internal replay evidence for later WM-0057 composition; retained rows
are serializable numeric/string enum data owned by sim-core.

Performance: normal scheduled mood updates visit only the current phase bucket
and actor-linked bounded thought/memory rows. Fact helpers read one actor's
fixed need lanes, an explicit environment projection, or an explicit health
condition view; they do not scan all actors, conditions, relationships, jobs,
or world cells. Metrics report generated/retained thought and memory counts,
evictions, expirations, mood update visits, and dirty backlog peak/final
backlog. Cross-actor eviction remains O(rows capacity) only at bounded capacity
pressure and then updates the exact evicted actor.

Security: no new dependencies, packages, Worker protocol messages, renderer
authority, preload surface, filesystem access, or schema entry points.

## Decisions and deviations

ADR-0008 is the controlling decision. WM-0054 implements only the
`MoodThoughtMemoryStore` owner slice and structured reason hooks. Relationship
events are not implemented here beyond generic source kind support for future
WM-0055 integration. The requested `pnpm test --filter m3-mood-thoughts`
command is unsupported by the existing test wrapper, so the direct Vitest file
and supported `sim-core` filter were run.

`m3-mood-thoughts.ts` exceeds the preferred 400-line file guideline because it
keeps one focused owner store, its bounded row lanes, trace store, metrics, and
hash helpers together, following the existing `m3-needs.ts` and `m3-health.ts`
M3 owner-store pattern. No unrelated abstraction or cross-package boundary was
introduced.

Reviewer finding `MSG-MQTBU7GT-B016FC` was valid. The original bounded global
row eviction reused the slot but only recalculated the new recipient actor.
The repair records the evicted actor before unlinking and updates that exact
actor's target/version as appropriate.
