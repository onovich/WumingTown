## Acceptance evidence

- WorkOffer mutations advance stored row/index versions once on success and
  not on failure. Reads and selections copy the selected row's stored owner and
  row versions plus the current index version into caller-owned outputs.
- Path results write only after the full route fits; a short output reports
  `path_output_capacity_exceeded` with `pathCellCount = 0` and leaves the route
  lane unchanged. The focused probe replaces all three legacy materializing
  MapGrid reads with throwing stubs and still resolves the exact legacy path.
- MapGrid caller-owned reads reset every output field for success, invalid
  index, non-neighbor, blocked terrain, wall, closed door and zero walk cost.
  Repeated reads preserve global/cell/chunk versions, dirty state, snapshot and
  canonical hash fields.
- Reservation preparation uses caller-owned typed lanes, validates headers,
  all five channels, pending duplicate/amount contention, ledger capacity and
  claim output capacity before one commit loop. Every tested failure preserves
  snapshot, version, active count and reset claim-id output.
- Ability output parity covers rebuild, cache hit, stale rebuild, denied
  threshold and invalid actor while bypassing the legacy allocating penalty
  result.
- The construction audit covers every new Into method, class helper and the
  free scalar/typed-lane helpers they call. No hot implementation constructs an
  object, array, typed array, closure or dynamic string. Existing allocating
  wrappers remain explicitly outside this proof.
- Scope/source audit found no DOM, React, Pixi, Electron, Node filesystem,
  real-clock, random, Promise/coroutine, protocol, Worker, Web, GameSession,
  save/load, dependency, content, PR-3 or release change.
