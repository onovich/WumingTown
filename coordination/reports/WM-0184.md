# WM-0184 Work report

## Outcome

`HaulingClaimFactsIndex` now exposes an O(1), allocation-free, caller-owned
`readMappingInto` seam for exact Hauling mapping revalidation. The read is keyed by
descriptor, work type, offer id, and opaque target id; it returns only Hauling-owned mapping
lanes and versions and does not read or infer Storage, ItemStack, or ReservationLedger facts.

## Files and public contracts changed

- `packages/sim-core/src/autonomy-claim-facts.ts`: added
  `HaulingClaimMappingReadInput`, `HaulingClaimMappingIntoOutput`, and
  `HaulingClaimFactsIndex.readMappingInto(input, output): void`.
- `packages/sim-core/src/autonomy-claim-facts.test.ts`: added exact success, key mismatch,
  missing/stale, output reuse/reset, zero-mutation, and unchanged snapshot/hash evidence.
- `packages/sim-core/src/owner-autonomy-adoption-closure.test.ts`: added the receiver-exact
  mapping-read root and fake-receiver rejection; reused one immutable TypeScript Program across
  project-root audits while keeping each audit queue/result isolated.
- `packages/sim-core/src/index.ts`: publicly exported the two caller-owned mapping read types.
- `coordination/tasks/WM-0184.json`: taskctl claim metadata, constrained quality-gate verdict,
  and mechanical formatting only.
- `coordination/reports/WM-0184.md`: this report.

Exact public signature:

```ts
readMappingInto(
  input: HaulingClaimMappingReadInput,
  output: HaulingClaimMappingIntoOutput,
): void;
```

The output contains `ok`, structured `reason`, all four query keys, source/destination slot ids,
source/destination interaction spot ids, mapping row version, and mapping index version. Every
call resets all output fields first. Missing/out-of-range rows return
`hauling_claim_descriptor_unmapped`; stale or malformed exact keys return
`hauling_claim_basis_stale`.

## Determinism, save, performance and security impact

- Determinism: no authoritative mutation or iteration-order change; the read uses one bounded
  direct slot lookup.
- Save: no snapshot, restore, hash, or schema changes.
- Performance: hot read is O(1), allocation-free, and does not scan foreign owners. Test-only
  TypeScript Program reuse removes repeated whole-project parsing without sharing audit state.
- Security/boundary: no new dependency or package-boundary bypass. Only explicit public input and
  output types were added at the package root.

## Decisions and deviations

  `readClaimFactsInto` remains the final composed validator for Storage, ItemStack, Reservation,
  and WorkOffer bases.
- No deviation from the task's allowed paths or architecture freeze was required.
