## Determinism, save, performance and security impact

Determinism: no runtime behavior changed. ADR-0017 protects fixed 30 TPS,
seeded streams, stable iteration, owner-store hashes, explicit job state and
Worker/headless parity as future gates. Projection basis fields make
tick/snapshot/hash coherence mechanically testable.

Save: no save format, public container, migration function, codec, platform
save UI or compatibility promise changed. The new docs define the future
GameSession save authority boundary and require fail-closed behavior or a
separate reviewed gate.

Performance: no benchmark code changed. The architecture requires dirty-key
projection, indexed WorkOffer/path selection, explicit caps, reliable-message
backpressure and a 100000 tick leak/conservation gate in WM-0166.

Security: no Electron preload, IPC, filesystem, network, telemetry, account,
signing, store or distribution surface changed. Owner gates remain unchanged.

Protocol: no protocol file changed in WM-0162. ADR-0017 approves a single
future envelope schema bump from 2 to 3 and GameSession projection v1 in
WM-0164. Existing message families and protocolVersion 1 remain; mixed-build
connections fail closed and legacy scenario projections cannot back the
default product route.

## Decisions and deviations

- Chose one `GameSessionRuntime` hosted by Worker/headless instead of extending
  catalogVersion scenario modes or the WM-0150 two-pawn playable slice.
- Rejected keeping schema 2 because it cannot carry integrated render and UI
  facts. Approved schema 3 plus projection v1 without adding message families.
- Created four proposed PR-1 tasks because the smallest safe tranche needs one
  sim-core producer, one sole protocol/Worker producer, one Web consumer and one
  gate task. Concurrent write-heavy width is one.
- Kept all implementation tasks `proposed`; none were claimed or promoted.
- No PR-2+ task packets were created.
- The expected task branch was not present locally or on `origin`; I created
  the ordinary branch in this repository and did not create a linked worktree.
- A pre-existing `coordination/thread-registry.json` diff was present before
  WM-0162 edits and is outside this task's allowedPaths; it is not part of this
  architecture deliverable.
