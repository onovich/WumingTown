# WM-0126 Work report

## Outcome

WM-0126 consolidates the reviewed M8 UI, responsive layout, localization,
visual identity, first-play, accessibility, content, data-mod, long-save,
performance and M0-M7 regression evidence into a single readiness matrix for
internal 1.0 review. The matrix preserves the existing platform verdicts of Web
`demo-only` and Windows unsigned `ready-for-controlled-external-test`, and it
keeps public release, store, signing, telemetry, account, paid-service and
public save-compatibility actions explicitly owner-gated.

The current readiness conclusion is a verified stop-sign evidence artifact
rather than a green closeout candidate: the required benchmark reruns fail the
unchanged 20 percent blocking threshold on repeated runs, so WM-0126 documents
the regression evidence for independent review and roadmap integration instead
of claiming M8 is closeout-ready.

## Decisions and deviations

- Added a dedicated roadmap matrix document instead of expanding the task report
  into the only authority. This matches the existing M7 pattern where the task
  report and roadmap matrix are separate but cross-referenced artifacts.
- Kept the M8 decision framed as internal 1.0 readiness evidence rather than
  public-release readiness because WM-0111, WM-0121, WM-0097 and WM-0110 all
  preserve the Web/Windows/public-save/public-service owner gates.
- The required performance and regression checks are run with `corepack pnpm`.
  `ci:local` and `bench` were pointed at temp artifact roots via
  `WM_ARTIFACT_DIR` to avoid rewriting reviewed tracked artifacts, and the
  `WM-0118` screenshot/json side effects from the E2E rerun were restored to
  `HEAD` before completion.
- `ci:local` needed a PATH correction on rerun because the script's nested bare
  `pnpm` initially resolved to 11.7.0 instead of the repo-required 11.8.0.
  The required gate was rerun without changing the repo or relaxing the check.

- WM-0126 does not approve public release, public Web launch, Windows public
  release, store submission, signing, telemetry, accounts, hosted services,
  paid services or public save compatibility.
- The current M8 content evidence is a scoped 1.0 package rooted in the
  accepted anomaly roster spec, faction/endgame owner slice, fail-closed
  content workflow and focused long-save gate. Any broader 1.0 authoring
  expansion still needs its own reviewed evidence.
- The repeated blocking benchmark regressions in `entity-store`,
  `m4-core-vertical-slice-long-run`, `map-dirty` and `spatial-index` are the
