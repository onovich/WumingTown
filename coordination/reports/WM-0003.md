## Determinism, save, performance and security impact

- Determinism: the smoke shell uses a static read model plus fixed Chromium viewport/device scale for screenshot generation; no authoritative simulation timing or randomness was introduced.
- Save: none. The task stays within web-shell presentation scope and does not implement save/persistence flows.
- Performance: renderer work stays in Pixi with chunk-based terrain drawing and entity markers; the React layer renders HUD/inspector only and does not create per-cell DOM.
- Security: none. No Electron work, preload surface, Node filesystem bridge, or privileged browser capability was added to the shell.
