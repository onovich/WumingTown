## Acceptance evidence

- Validates all required viewports:
  Web and desktop responsive matrix tests iterate the full seven-viewport set
  and record DOM-layout metrics in `web-responsive-layout.json` and
  `desktop-responsive-layout.json`.

- Checks no core UI clipping, incoherent overlap, missing key information or
  unreachable scroll regions:
  The E2E assertions verify viewport-bounded main-menu surfaces, reachable
  first-play guidance, reachable player HUD sections, visible world canvas,
  reachable tall inspector/storage regions, and explicit debug-overlay
  separation.

- Runs checks in `zh-CN` and `en`:
  The matrix covers both locales automatically for the default player surfaces.
  Reviewer screenshots are intentionally a curated subset, but the JSON
  artifacts and automated assertions cover the full bilingual matrix. This is
  not a verified limitation.

- Produces reviewer-suitable screenshot or DOM-layout artifacts:
  The JSON artifacts include selector metrics and document overflow results for
  every matrix case. The screenshot subset provides direct visual samples for
  start surface, in-game HUD and debug overlay.

- Does not weaken existing E2E or benchmark gates:
  Existing smoke coverage remains intact. The new work only extends E2E
  evidence and adds a stable HUD test selector.

## Notes on scope and limitations

- `coordination/tasks/WM-0118.json` was updated to allow
  `coordination/artifacts/WM-0118/**` so the reviewer artifacts can live in the
  task branch without violating task path policy.
- Developer diagnostics remain English-only inside the explicit debug overlay.
  This matches the M8 scope amendment allowance because the default player UI
  stays localized and diagnostics stay separate from it.
- No verified responsive limitation remains open for the required viewport set.
  Small-height cases rely on reachable scroll regions for tall inspector and
  storage panels, which is consistent with the acceptance criteria.
