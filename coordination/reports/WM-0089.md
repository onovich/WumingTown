# WM-0089 Work report

## Outcome
WM-0089 verifies the M6 Web behavior when `SharedArrayBuffer` is unavailable: the browser Worker smoke path runs in non-cross-origin-isolated Chromium, selects the existing structured-clone / Transferable snapshot fallback, and keeps simulation authority inside the Worker with read-only projection payloads matching the M5 headless replay hashes. No public protocol family, protocol version, schema version, UI authority path or M7 hosting assumption was added.

## Files and public contracts changed
- `packages/sim-worker/src/sharedarraybuffer-fallback.ts`: added a small transport gate helper that classifies SAB availability and records the fallback cap impact without moving authority out of the Simulation Worker.
- `packages/sim-worker/src/sharedarraybuffer-fallback.test.ts`: covers the SAB-available and non-isolated fallback decisions.
- `packages/sim-worker/src/worker-smoke.e2e.test.ts`: extends the browser Worker harness to capture page and dedicated Worker runtime facts, then proves the M5 read-only projection path with SAB unavailable.
- `tools/test-runner.mjs`: narrow owner-approved routing for the required `pnpm test --filter worker-smoke` unit check.
- `docs/05_tech/02_worker_protocol.md`: records that WM-0089 does not change Worker protocol families, versions or schema.
- `docs/05_tech/06_platform_matrix.md`: records the Web SAB fallback status and conservative product-gate impact; the file was normalized to ASCII Markdown while preserving the platform matrix content.
- `coordination/reports/WM-0089-plan.md`: execution plan for the task.
- `coordination/tasks/WM-0089.json`: claim state and the narrow `tools/test-runner.mjs` exception history entry.

## Acceptance evidence
- Web runtime behaves correctly without SharedArrayBuffer: `pnpm test:e2e --filter worker-smoke` runs the real browser module Worker path and asserts page and dedicated Worker `crossOriginIsolated` are `false` with `SharedArrayBuffer` type `undefined`.
- Transferable snapshot fallback preserves authority and read-only projections: the new gate returns `snapshotTransport: "transferable-snapshot"`, `authorityOwner: "simulation-worker"` and `projectionPolicy: "read-only"` for the non-isolated browser runtime.
- Cross-origin isolation assumptions are recorded: `docs/05_tech/02_worker_protocol.md` and `docs/05_tech/06_platform_matrix.md` state that COOP/COEP remains required before a future SAB transport can be considered.
- Fallback performance or cap impact is measured or recorded: the e2e harness records positive browser Worker elapsed time, and the platform matrix records the conservative 3x/lower-cap or demo-only decision path until product-scale evidence exists.
- M0-M5 regression coverage: the browser Worker M5 checkpoint sequence still matches the headless final world hash, read-model hash and `readOnly: true` projection flags.
