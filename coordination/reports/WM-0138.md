# WM-0138 Work report

## Outcome

Implemented one minimum playable Web shell interaction chain: the player can select a lamp-relevant target, use the HUD command bar to queue `prioritize-lamp-work`, and see localized queued feedback backed by a machine-readable, explicitly non-authoritative shell-local adapter record.

## Files and public contracts changed

- `packages/ui-react/src/shell-store.ts`: added `ShellPlayableActionState` and `ShellCommandActions` for traceable shell-local player actions.
- `packages/ui-react/src/shell-hud.ts`: replaced the primary command placeholder with a selectable lamp-priority command, queued feedback panel, command state attributes, and adapter evidence attributes.
- `packages/ui-react/src/localization.ts`: added en/zh-CN command and feedback copy for the playable local action.
- `packages/ui-react/src/shell-read-model-localization.ts`: localized `Action queued ...` input feedback.
- `apps/web/src/shell-bootstrap.ts`: added the Web shell local adapter that records queued lamp-priority action state without sending unreviewed Worker commands or mutating Simulation Worker authority.
- `apps/web/src/web-storage-gate.ts`: clears session-local playable action state when loading a save envelope, because the adapter state is intentionally not part of save data.
- `apps/web/src/web-shell.e2e.test.ts`: added en and zh-CN coverage for select -> command -> queued feedback -> debug payload.
- `packages/ui-react/src/shell-hud.test.ts`: added HUD render coverage for the queued local action and updated command label expectations.
- `packages/ui-react/src/index.ts`: exported the new shell action types.
- `coordination/reports/WM-0138-plan.md`: execution plan recorded before product edits.

## Acceptance evidence

- Player can select a lantern/resident/object and issue a meaningful HUD action: `apps/web/src/web-shell.e2e.test.ts` now selects `lantern-keeper-shen`, clicks `player-command-lamp`, and verifies the command state changes from `ready` to `queued`.
- Action reaches existing protocol/simulation path or reviewed local adapter: implemented `wm0138-web-local-playable-adapter` as shell-local state only; public Worker protocol remains unchanged.
- Read-model or shell state visibly changes with structured reason: HUD renders `player-action-feedback` with `data-action-authority="shell-local-adapter"`, `data-reason-code="wm0138.local_adapter.lamp_priority"`, target id, adapter id, and command id; debug payload exposes the same action.
- No fake world authority: the adapter only updates `ShellState.playableAction` and `lastInputLabel`; it does not modify sim-core, save schema, Worker protocol, or authoritative read-model data. Loading a save clears this session-local action to prevent stale non-saved feedback.
- zh-CN and en E2E coverage: `queues a traceable local lamp-priority action in en and zh-CN` passed for both locales.

## Determinism, save, performance and security impact

Determinism: unchanged for authoritative simulation; the new command id uses a per-shell-session UI sequence only for local adapter evidence.

Save: no save schema or public compatibility change; `web-storage-gate.ts` continues to save only the existing gate envelope fields and clears the session-local action on load.

Performance: no simulation hot path touched; HUD render work is bounded to current shell state and selected entity.

Security: no new dependency, IPC, filesystem, or Worker protocol exposure; adapter remains browser shell state.

## Decisions and deviations

- Chose the documented local adapter route because `packages/sim-protocol` currently supports only `Noop` and `Echo`, and WM-0138 forbids unreviewed public Worker protocol changes.
- Required command `corepack pnpm test --filter web-shell` is unsupported by the repo test runner because `pnpm test` invokes unit mode. Effective Web coverage was run with `node tools/test-runner.mjs e2e --filter web-shell`.
- First review requested one fix: because `playableAction` is not saved, loading a save must not preserve stale queued action feedback. Fixed in `apps/web/src/web-storage-gate.ts` and covered in the OPFS round-trip E2E.

## Remaining risks / follow-up tasks

- The action is intentionally non-authoritative. A later reviewed gameplay/protocol task must replace or connect this adapter to authoritative Simulation Worker command handling.
- Only the lamp-priority chain is playable; chronicle and inspect command slots remain placeholders by scope.
