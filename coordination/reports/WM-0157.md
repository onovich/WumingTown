# WM-0157 Architecture report

## Outcome

No legal reviewed public browser Worker/session bridge exists for `apps/web`
today. WM-0152 must remain blocked until the new precursor task `WM-0157`
lands. I created ADR-0013 and the ready task packet for the narrow bridge:
root-export a browser session API from `@wuming-town/sim-worker`, explicitly
allow `apps/web` to consume that package root, and keep
`packages/sim-worker/src/browser-worker-entry.ts` internal.

## Verdict for WM-0152

WM-0152 cannot resume yet. The exact legal path does not currently exist.

Current inspected state:

- `apps/web/package.json` allows `@wuming-town/foundation`,
  `@wuming-town/platform`, `@wuming-town/renderer-pixi`,
  `@wuming-town/sim-protocol` and `@wuming-town/ui-react`; it does not allow
  `@wuming-town/sim-worker`.
- `apps/web/src/shell-bootstrap.ts` creates
  `createReviewedPlayableProjectionSession()`, which is WM-0151 projection
  playback rather than Worker authority.
- `packages/sim-worker/package.json` exports only `"."`.
- `packages/sim-worker/src/index.ts` exposes in-process Worker helpers and
  command handling, but no browser session client consumed by Web.
- `packages/sim-worker/src/browser-worker-entry.ts` is a package-internal Worker
  entry.
- `packages/platform/src/index.ts` exposes platform save/mod ports and M6
  diagnostic helpers, not a Simulation Worker session adapter.
- `tools/boundary-check.mjs` rejects cross-package subpath imports, so a direct
  `@wuming-town/sim-worker/browser-worker-entry` import would still violate the
  current boundary contract.

## Decision

Choose the narrow root-export bridge:

- `@wuming-town/sim-worker` owns a public browser session client or equivalent
  root-exported entry URL/factory.
- `apps/web` may add `@wuming-town/sim-worker` as a dependency and allowed
  internal dependency only after WM-0157.
- Web imports only `@wuming-town/sim-worker`.
- `browser-worker-entry.ts` stays internal to the Worker package.
- The bridge is transport/read-model delivery only; it is not UI authority.

Rejected or deferred alternatives:

- Deep import `packages/sim-worker/src/browser-worker-entry.ts`: rejected
  because it violates package boundaries.
- Keep WM-0151 projection playback: rejected for WM-0152 because it is not
  authoritative.
- Put the adapter in `@wuming-town/platform`: deferred because it would pull
  Worker/sim-core ownership into a generic platform package.
- Create a new `@wuming-town/sim-session` package: deferred because it adds
  workspace topology before the first unblock.

## Determinism, save, performance and security impact

Determinism: no runtime behavior changed. ADR-0013 requires future bridge work
to send deterministic protocol messages only and to keep Worker/headless
authority intact.

Save: no public save compatibility or migration promise was added. WM-0157 must
leave in-progress playable command persistence unsupported or fail-closed with
structured reasons until a separate versioned save task is approved.

Performance: no runtime path changed. ADR-0013 requires event-driven Worker
delivery, explicit lifecycle, no hidden singleton scans, and no reliable
message drops behind render backpressure.

Security: no Electron, preload, filesystem, telemetry, account, store, release
or signing surface changed. The decision forbids exposing generic internals or
using UI-owned authority.

## Migration and rollback

Migration path:

1. Implement WM-0157 in `@wuming-town/sim-worker` as a package-root browser
   session bridge.
2. Add the reviewed `apps/web -> @wuming-town/sim-worker` manifest boundary.
3. Prove real browser Worker communication through the public root bridge.
4. Resume WM-0152 and replace the WM-0151 projection harness with a thin Web
   command/read-model adapter.

Rollback before WM-0152 consumes the bridge is deleting the new root API and
removing the Web dependency edge. Rollback after WM-0152 consumes it must return
WM-0152 to an explicit blocked state or to clearly labeled projection playback;
it must not deep-import internals or invent UI-local authority.

## Remaining risks / follow-up tasks

- WM-0157 must choose final public symbol names and report them so WM-0152 can
  resume without guessing.
- Web save/load for in-progress playable command state remains a future
  reviewed task, not part of this unblock.
- If future Electron wants the same bridge through platform ports, that should
  wrap the root `sim-worker` API rather than duplicating Worker protocol logic.
