# WM-0157 Work report

## Outcome

Implemented a package-root browser Simulation Worker session bridge for
`@wuming-town/sim-worker` and added the reviewed `apps/web` dependency edge to
consume it without deep imports. The bridge owns only Worker transport,
sequence/lifecycle helpers and read-model message delivery; authority remains
inside the Simulation Worker/headless runtime.

## Files and public contracts changed

- `packages/sim-worker/src/browser-session.ts`: added the public browser Worker
  session client, default module Worker factory, reliable message subscription,
  explicit lifecycle, and WM-0150 playable scenario initialization helper.
- `packages/sim-worker/src/index.ts`: root-exported the browser session API.
- `packages/sim-worker/src/worker-smoke.e2e.test.ts`: moved Chromium smoke
  execution through a Vite module that imports `@wuming-town/sim-worker` from
  the package root, and added WM-0150 `PrioritizeLampWork` /
  `QueueSimpleBuild` browser Worker coverage.
- `apps/web/package.json`: added the reviewed root dependency and
  `wumingBoundary.allowedInternalDependencies` entry for
  `@wuming-town/sim-worker`; no `@wuming-town/sim-core` dependency was added.
- `apps/web/src/simulation-worker-session.ts`: added a focused Web adapter that
  re-exports only transport helpers from the package-root bridge.
- `apps/web/src/simulation-worker-session.test.ts`: verifies Web can start the
  WM-0150 scenario and send command batches through the bridge without local
  authority.

## Determinism, save, performance and security impact

Determinism: no sim-core deterministic logic changed. Browser bridge assigns
monotonic integer message sequences and sends typed protocol envelopes; Worker
authority remains unchanged.

Save: no public save compatibility or migration promise is added. Web may send
`requestSave`/`loadSession` transport messages, but in-progress playable command
persistence remains unsupported until a separate versioned save task is
approved.

Performance: no pawn think loop or world scan was added. The new session
client only dispatches received Worker messages to subscribed listeners in
stable order. Worker-smoke WM-0150 evidence still reports bounded candidate and
path behavior from the authoritative runtime.

Security: no Electron, preload, filesystem, telemetry, accounts, signing,
public release, or paid-service surface changed. Browser construction is local
module Worker transport only.

## Decisions and deviations

- Followed ADR-0013 option 5: expose the browser session client from the root
  of `@wuming-town/sim-worker` and allow `apps/web -> @wuming-town/sim-worker`.
- Kept `packages/sim-worker/src/browser-worker-entry.ts` internal; no
  package-subpath export or Web deep import was added.
- The Web adapter does not replace the WM-0151 projection harness in
  `shell-bootstrap.ts`; product HUD/placement/localization wiring remains for
  WM-0152.
- No Spark/rapid-implementer lane was used; this task changes a public Worker
  bridge and package boundary contract.

## Remaining risks / follow-up tasks

- WM-0152 still needs to wire product UI commands to this bridge and build
  commands from reviewed Worker/read-model basis data.
- Current Worker `UiDelta` still carries compact summaries, not a full public
  playable read-model object for Web product UX.
- Public save/load for playable command state remains unsupported by design.
- `coordination/thread-registry.json` was already dirty from
  project-director thread registration before implementation; it was preserved
  as control-plane state.
