# WM-0165 Work report

## Outcome

The default Web gameplay route now renders the authoritative PR-1 GameSession
Worker projection as its only town truth. It requests exact schema-v3
`{ kind: "game_session", version: 1 }`, waits for exact Ready, pairs only
coherent RenderSnapshot/UiDelta bases, and observes normal time through the
WM-0164 Worker scheduler. It does not send PlayerCommandBatch, advance, or
drain requests as a clock and does not fall back to a static fixture.

The changes-requested repair is complete. Resource projection rows map
losslessly to the additive presentation kind `resource`; Pixi draws a distinct
resource marker and selection outline; and the HUD renders localized resource
semantics in en and zh-CN. At 1424x861, real Playwright mouse input selects an
uncovered resident and then an uncovered resource, and the inspector displays
the exact same-basis available/reserved/total tuple.

### Selection identity

- `selectedEntityId: undefined` means first coherent frame and may default to
  the first resident.
- `selectedEntityId: null` means an explicit empty selection and remains empty.
- A non-null explicit id absent from a newer coherent frame now returns empty;
  it never jumps to the first resident.
- Focused coverage switches rapidly from resident detail to another resident
  and to a resource while the old detail is still present. Identity mismatch
  leaves detail fields empty until the matching UiDelta arrives. An unavailable
  id clears selection.
