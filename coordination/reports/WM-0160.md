# WM-0160 Architecture Report

## Outcome

Created WM-0160 as the narrow public architecture precursor for WM-0152's
remaining blocker: a package-root `@wuming-town/sim-worker` playable
drain/terminal wait helper that advances through explicit target ticks and
resolves on authoritative `UiDeltaPayload.playable` state.

This is not product UI work. WM-0160 is scoped to the public Worker/session
surface, focused tests, a thin Web adapter if needed, and Chromium worker-smoke
coverage.

## Blocker Verification

Current public facts after WM-0157/WM-0158/WM-0159:

- Web can construct a public browser Simulation Worker session.
- Web can send WM-0150 commands through `PlayerCommandBatch`.
- Web can receive reliable `CommandResult` messages.
- Web can read `UiDeltaPayload.playable`.
- Web can request deterministic explicit advance via the public helper.

The missing contract is awaitable completion/drain. The current
`BrowserSimulationWorkerSession` has callbacks and an explicit advance method,
but no public root helper to await a new playable `UiDelta` or to drain known
command ids until terminal/quiescent projection state. WM-0152 WIP therefore
uses a browser timer pump and debug/projection polling to decide when to stop.

I reproduced the reported gate failure:

```bash
corepack pnpm exec vitest run apps/web/src/web-shell.e2e.test.ts -t "shows authoritative job marker, pawn motion, work progress, and completion for lamp and build commands"
```

Result: FAIL after `120000ms`.

```
Test Files  1 failed (1)
Tests       1 failed | 12 skipped (13)
Error: Test timed out in 120000ms.
```

This confirms WM-0152 remains blocked on a public session/protocol contract, not
on another product UI implementation step.

## Decision

Added ADR-0016: Public playable Worker drain and terminal wait helper.

Chosen direction:

- expose a public root `@wuming-town/sim-worker` browser session helper;
- use existing explicit advance requests and `PlayerCommandBatch`;
- wait for existing authoritative `UiDeltaPayload.playable`;
- derive terminal state only from protocol-owned `PlayableProjectionV1`;
- keep at most one in-flight advance target per helper invocation;
- return posted messages plus the final authoritative `UiDelta` and projection;
- avoid new Worker message families, timers, hidden singletons and Web-owned
  terminal authority.

Deferred:

- autonomous `SetSpeed`/`Pause` clock loops;
- new top-level Worker drain/terminal message family;
- product HUD/shell/Pixi implementation;
- save/load persistence for in-progress playable command state.

## WM-0152 Resume Path After WM-0160

WM-0152 should replace debug hot-poll terminal waits with the thin Web adapter
around the new root helper, for example:

```ts
import {
  advanceWebPlayableWorkerScenarioToTick,
  drainWebPlayableWorkerCommands,
  readWebPlayableProjection,
} from "./simulation-worker-session";
```

Product code should still render only `readWebPlayableProjection(...)` and
reliable `CommandResult` output. It must not import `sim-core`, deep-import the
Worker, parse summaries, or treat diagnostic debug payloads as gameplay state.

## Acceptance Summary

WM-0160 requires:

- public root helper/session method;
- no new message family unless ADR-0016 is superseded;
- no Web `sim-core` or deep imports;
- terminal state derived only from `PlayableProjectionV1`;
- explicit target ticks and one in-flight advance per helper invocation;
- cancel/destroy cleanup;
- real Chromium worker-smoke proving lamp/build terminal projection state;
- reliable messages still delivered separately;
- no product UI, save compatibility, telemetry or release promise.

## Save, Determinism, Performance and Security Stance

Determinism: drain helpers use explicit integer target ticks and authoritative
Worker `UiDelta` messages. Real time is not simulation authority.

Save: no public save compatibility, migration or in-progress playable command
persistence is introduced.

Performance: the design replaces hot debug polling with message-driven waits,
keeps one in-flight advance target per helper invocation, and must clean up
subscriptions.

Security/release: no Electron, preload, filesystem, telemetry, accounts, paid
services, signing, store or public release surface changes.
