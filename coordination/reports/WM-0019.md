# WM-0019 Work report

## Outcome

Implemented the M1 authoritative deterministic map grid in `sim-core` with typed
numeric storage for terrain, occupancy, walk cost, region id, room id,
per-cell versions, chunk versions and chunk dirty metadata. Dirty chunks are
queued once, drained through an explicit budgeted API, and exposed through
stable snapshot/hash hooks for future save/replay integration.

## Acceptance evidence

- Contiguous numeric storage: `MapGridSnapshot` uses `Uint16Array`,
  `Int32Array`, `Uint32Array` and `Uint8Array`; verified by
  `packages/sim-core/src/map-grid.test.ts`.
- Authoritative map fields: `MapGrid` owns terrain, occupancy, walk cost,
  region id, room id, cell version, chunk version and chunk dirty lanes.
- Dirty queues local and budgeted: `processDirtyChunks(budget, output)` drains
  into caller-owned typed buffers; tests verify unchanged writes do not grow
  queues.
- Stable hash/snapshot hooks: `createSnapshot()` clones authoritative typed
  lanes, and `createMapGridHashFields()` hashes lanes in fixed order; tests
  verify canonical field-order stability.
- Scope boundaries: no DOM, React, Pixi, Electron, Node filesystem, real time,
  ambient randomness, pathfinding, reservations, jobs or renderer code in
  `sim-core` map files.

## Determinism, save, performance and security impact

Determinism: map coordinates, values, versions, dirty queue order and hash
fields are integer and stable-order. No ambient randomness or wall-clock inputs
enter `sim-core`.

Save/replay: introduced authoritative-data-only `MapGridSnapshot` and canonical
hash fields. This is not a save container promise and does not implement
ADR-0005 persistence.

Performance: hot map mutation and dirty drain paths use typed arrays and loops;
no normal-path full-world scan is required to drain dirty chunks. Benchmark
reports changed cells, dirty queue peak, processed chunks, remaining backlog,
checksum and map hash.

Security/boundaries: `sim-core` remains free of DOM, React, Pixi, Electron and
Node filesystem imports.

## Decisions and deviations

- Cites `coordination/decisions/ADR-0003.md`: map storage is an owner store in
  `sim-core`; occupancy is a numeric authoritative lane but does not implement
  location ownership or spatial indexes, which remain WM-0020 scope.
- Cites `coordination/decisions/ADR-0006.md`: walk cost uses integer
  `PathCostMilli` with `1000` neutral cost, dirty queues use deterministic
  chunk order, and hash input order is stable.
- Did not use Spark/rapid-implementer; work was performed by
  `simulation-engineer` in the WM-0019 worktree.

## Remaining risks / follow-up tasks
