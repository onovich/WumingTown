# WM-0040 Work report

## Outcome

Added a focused M2 work/logistics save/replay resume harness for
`m2.work_logistics.lantern_yard.v1`. The harness saves at tick 6000, validates
the envelope before load succeeds, rebuilds derived indexes as non-authoritative
read models, and proves resumed replay matches uninterrupted replay to tick
20000. No platform save UI, public save compatibility promise, dependency or
schema migration was introduced.

## Files and public contracts changed

- `packages/sim-core/src/m2-save-replay.ts`: focused M2 save envelope,
  load/resume validation, checkpoint replay and comparison helpers.
- `packages/sim-core/src/m2-save-replay.test.ts`: focused save/resume,
  fail-closed validation, projection isolation, divergence and command id tests.
- `packages/sim-core/src/index.ts`: exports the focused M2 scenario and
  save/replay diagnostics helpers so tools consume sim-core through the public
  package entry point.
- `tools/test-runner.mjs`: added the literal `m2-save-replay` unit filter.
- `tools/sim-replay-test.mjs`: added explicit replay diagnostics argument
  parsing for `--scenario m2-work-logistics`.
- `tools/sim-replay-diagnostics-lib.mjs`: added the M2 replay diagnostics path
  and WM-0040 artifact paths without changing the default M1/general run.
- `docs/02_systems/13_save_replay_migration.md`: documented the focused M2
  envelope and validation boundary.
- `docs/05_tech/05_testing_observability_ci.md`: documented the focused test
  filter and replay CLI.
- `coordination/tasks/WM-0040.json`: task claim and completion state.

The `index.ts` exports are a focused diagnostics entry point required by the
package-boundary rule; they do not add a public save container, platform save
UI, cross-version compatibility promise or migration guarantee.

## Acceptance evidence

- M2 interrupt save/load/resume parity: `pnpm test --filter m2-save-replay`
  passes; `pnpm sim:replay-test -- --scenario m2-work-logistics` reports final
  world hash `0xf2a38685` and read-model hash `0xfc786747`.
- Load validates before success: focused tests mutate format version, scenario
  id, sorted job records, owner handles, integer lanes, command-log hash and
  projection hash; all fail closed with structured reasons.
- Derived indexes rebuild: load returns `work-offers`, `path-caches`,
  `reservations` and `read-models`; none are persisted as authoritative cache
  payloads.
- Diagnostics name failure context: comparison helpers return scenario id,
  seed, first divergent tick and artifact paths; the CLI prints those fields on
  failure.
- Scope stays focused: no platform save UI, package manifest, lockfile, Worker
  protocol, schema dependency or public save compatibility change was made.

## Determinism, save, performance and security impact

Determinism: all replay checkpoints are generated from explicit seed and tick
inputs; no ambient random/time is introduced. Save: focused M2 envelope only,
with magic/version/scenario/record/projection validation and no public
compatibility promise. Performance: save size is 4370 bytes in the focused
diagnostic; no hot tick path is changed. Security/runtime boundaries: sim-core
remains free of DOM, React, Pixi, Electron, Node filesystem, real time and new
runtime dependencies.

## Decisions and deviations

- ADR-0007 remains the controlling M2 save/replay architecture gate; this task
  does not change public save sections, migration behavior or Worker protocol.
- First review requested changes because `tools/sim-replay-diagnostics-lib.mjs`
  imported M2 sim-core internals directly. The fix exports the focused helpers
  through `packages/sim-core/src/index.ts` and updates the tool to import only
  from that public entry point.
- `m2-save-replay.ts` follows the existing single-file M1 harness shape. The
  file is over the preferred 400-line guidance because the task allowed a
  focused harness file but did not authorize a new helper module; the code stays
  non-hot and local to this task boundary.
- CLI artifacts under `coordination/artifacts/WM-0040/` are generated evidence
  and are not committed.
