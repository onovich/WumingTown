## Determinism, save, performance and security impact

Determinism: ADR-only. The proposal preserves deterministic resume by requiring
versioned random stream state, command-log tail continuity, rebuilt indexes, and
pre-resume invariant validation.

Save: Defines a Proposed M1 container contract only. It does not implement
serialization, choose a byte codec, add a dependency, expose save UI, or promise
compatibility beyond M1.

Performance: No runtime impact in this task. The ADR requires later benchmarks
to measure total/per-section size, validation, migration, rebuild time, and
first-resumed-tick hash comparison.

Security: External save data is treated as untrusted. The ADR requires boundary
validation of section directory, checksums, versions, scalar ranges, declared
counts, ids, and references before `WorldState` mutation.
