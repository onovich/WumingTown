# WM-0153 Work report

## Outcome

Implemented the first-play guidance, localization, responsive, and accessibility repair around the verified WM-0152 playable lamp/build command chain. The player-facing start surface and HUD now describe the real command path: enter town, select the East Market Lantern Post or map target, issue the authoritative lamp command, enter Build mode, hover valid or invalid blueprint tiles, place the available blueprint command, and read accepted/rejected command, job, pawn, progress, completion, or blocked-reason feedback. Settings and diagnostics remain secondary surfaces.

## Files and public contracts changed

- `packages/ui-react/src/localization.ts`: rewrote first-play, onboarding, save/load stance, lamp/build command, and zh-CN/en copy so it matches the real Worker command surface and does not advertise unavailable controls.
- `packages/ui-react/src/shell-main-menu-surface.ts`: changed the first-play action list from generic/static guidance to the concrete lamp chain, build chain, and structured feedback path.
- `packages/ui-react/src/shell-read-model-localization.ts`: added player-visible localization coverage for WM-0152 projection strings, including structure summaries, job states, pawn/activity states, alerts, resources, target labels, and blocked reasons.
- `packages/ui-react/src/shell-hud.ts`: localized structured task/reason feedback, added invalid build-hover feedback, and compacted medium-layout command feedback so 1280x720 keeps map focus, action bar, active feedback, and inspector usable.
- `packages/ui-react/src/shell-hud.test.ts`: updated first-play and zh-CN HUD assertions for authoritative command wording and no stale local-action guidance.
- `apps/web/src/web-shell.e2e.test.ts`: added WM-0153 assertions for first-play copy, zh-CN/en consistency, focused responsive viewports, focus affordances, real lamp/build feedback, and invalid blueprint hover feedback.
- `coordination/tasks/WM-0153.json`: task ownership state from workflow claim was preserved.

No Simulation Worker, `sim-core`, public protocol, save format, release, telemetry, account, paid-service, store, or signing path was changed.

- Determinism: unchanged. UI still consumes the public WM-0152 Worker projection and does not synthesize command, job, pawn, resource, or build authority.
- Save/load: explicitly unsupported for authoritative command runtime restore. Continue copy now states that local shell evidence can be restored, but in-progress/completed command runtime is not restored from storage.
