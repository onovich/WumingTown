## Determinism, save, performance and security impact

Determinism: the new test and benchmark use existing M3 deterministic scenario,
replay and hash APIs. The benchmark validates replay hash parity through
`100000` ticks and save/resume parity from tick `12000` to `36000`.

Save: the task measures existing M3 focused save/load rebuild behavior and
rebuilt surface count. It does not alter save format, schema, migration,
compatibility or platform save behavior.

Performance: the baseline adds an M3 median of `24.456ms` with unchanged
`10` percent warning and `20` percent blocking thresholds. The final artifact
recorded M3 median `25.073ms`, save/load rebuild time `0.815ms`, Worker
projection bytes `1747` and status `OK`.

Security/authority: UI, Worker projection consumers, Pixi, React, Electron and
platform surfaces remain read-only. The benchmark consumes sim-core public
exports only and adds no dependency or runtime authority.

## Decisions and deviations

- ADR-0008 was followed: M3 benchmark evidence covers ordinary-life needs,
  health/ability, medical, mood/relationship, save/load, Worker projection size
  and deterministic hashes while preserving thresholds.
- The initial `qa-performance` subagent
  `019effc9-f0d8-7862-a21d-ee5c4222f8a9` was dispatched but reported no
  retained implementation diff and no running commands. The project-director
  thread completed the scoped implementation in the WM-0059 task worktree;
  independent reviewer verification is still required.
- `pathNodeCount` is derived from the existing M3 exact path result, where the
  fixture's successful path has `nodeExpansions = 2`. No product pathing code
  was changed to create extra measurement surfaces.
- Reviewer `019effdd-2a63-75b2-b559-9b86d3fee8c0` requested changes for stale
  medical offer coverage and range-only drift checks. The follow-up exposes the
  existing medical stale-basis reject metric in M3 summary end state and makes
  tests/benchmark invariants compare exact stable condition, mood and
  relationship values.
- During main integration, two broad `pnpm bench` runs failed under transient
  machine slowdown before the final pass. The M3 baseline median was refreshed
  from the main artifact generated at `2026-06-25T18:00:03.186Z`; warning and
  blocking thresholds remained `10` and `20`, and no invariant mismatch was
  introduced.
