# WM-0004 Work report

## Outcome

WM-0004 delivers a sandboxed Windows Electron shell that wraps the existing web client, exposes only typed placeholder platform ports through preload, builds a portable `win-unpacked` desktop artifact, and verifies both development and packaged launch paths with Playwright.

## Files and public contracts changed

- `apps/desktop-electron/package.json`: add Electron runtime pin and desktop build/package scripts.
- `apps/desktop-electron/package-dir.ts`: portable Windows unpacked packager that reuses installed Electron or downloads the pinned official ZIP into a local cache.
- `apps/desktop-electron/src/main.ts`: BrowserWindow creation with `nodeIntegration: false`, `contextIsolation: true`, `sandbox: true`, and navigation lockdown.
- `apps/desktop-electron/src/preload.cjs`: narrow contextBridge surface exposing only typed placeholder `host`, `saveStore`, and `mods` ports.
- `apps/desktop-electron/src/desktop-shell.e2e.test.ts`: Playwright Electron smoke coverage for dev-server and packaged launches.
- `apps/desktop-electron/tsconfig.build.json`: emit-only config for the Electron main entry without polluting workspace `src/` trees.
- `apps/web/index.html`, `apps/web/vite.config.ts`, `apps/web/src/shell-bootstrap.ts`, `apps/web/src/web-shell.e2e.test.ts`, `apps/web/package.json`, `apps/web/tsconfig.json`: CSP/build/test adjustments so the same renderer works in browser and packaged Electron flows.
- `packages/platform/src/index.ts`: typed platform bridge contract, structured unavailable results, and browser fallback resolver.
- `packages/renderer-pixi/src/pixi-world-renderer.ts`: explicit `pixi.js/unsafe-eval` opt-in required for CSP-constrained rendering environments.
- `package.json`, `pnpm-lock.yaml`, `pnpm-workspace.yaml`, `eslint.config.mjs`, `tools/test-runner.mjs`, `tsconfig.typecheck.json`: root build/test/lint wiring for desktop packaging and smoke coverage.
- `coordination/decisions/ADR-0002.md`: WM-0004 dependency and packaging review addendum.

## Acceptance evidence

- `nodeIntegration` disabled and `contextIsolation` / `sandbox` enabled:
  - `apps/desktop-electron/src/main.ts` hard-codes those BrowserWindow flags.
  - `apps/desktop-electron/src/desktop-shell.e2e.test.ts` asserts `window.process` and `window.require` are absent while the preload bridge is present.
- Preload exposes only typed placeholder platform ports:
  - `packages/platform/src/index.ts` defines the typed contract.
  - `apps/desktop-electron/src/preload.cjs` exposes only `host`, `saveStore`, and `mods`, each returning structured unavailable results.
  - Desktop smoke asserts `saveStore.list()` returns the placeholder unavailable payload.
- Development and packaged smoke builds launch on Windows:
  - `pnpm test:e2e` passed with 8/8 tests, including both desktop dev-server and packaged launch cases.
  - `pnpm build:desktop` produced `dist/desktop/win-unpacked/WumingTown.exe`.
- Security checklist has no unresolved high finding:
  - `pnpm audit --json` reports `high: 0`, `critical: 0`.
  - No generic `fs`, `shell`, or arbitrary IPC bridge exists in preload.
  - BrowserWindow navigation is locked down in `apps/desktop-electron/src/main.ts`.

## Determinism, save, performance and security impact

- Determinism: none to simulation authority. Changes stay in shell/build layers and do not touch `sim-core`.
- Save: no real persistence behavior yet. Only typed placeholder ports were introduced; they return structured unavailable results.
- Performance: no known simulation regression. Desktop packaged build emits a large renderer chunk warning from Vite/Pixi bundling; this is tracked as a follow-up optimization, not a correctness failure.
- Security: improved relative to the previous skeleton. Renderer runs with `nodeIntegration: false`, `contextIsolation: true`, `sandbox: true`; preload exposes no generic filesystem, shell, or arbitrary IPC surface; audit shows no unresolved high/critical vulnerability.

## Decisions and deviations

- ADR: `coordination/decisions/ADR-0002.md` now includes the WM-0004 Electron dependency and packaging review.
- Rejected `electron-builder` before completion. It was temporarily evaluated, but removed because the accepted `win-unpacked` requirement did not justify the extra packaging dependency and its unrelated external tool downloads.
- Final packaged artifact is a portable unpacked directory built by repository code in `apps/desktop-electron/package-dir.ts`, which is the approved equivalent for this task's Windows smoke requirement.

## Remaining risks / follow-up tasks

- Desktop packaging is currently Windows-only and portable-only; installer UX, signing, updater flow, and icons are not part of WM-0004.
- The packager downloads the official Electron ZIP on cache miss. This is explicit and cached, but offline-first distribution and mirrored artifact policy may need a later task.
- The renderer build still emits a large chunk warning; if web/desktop asset size becomes a budget issue, split or lazy-load non-critical Pixi/React code.
- Desktop smoke helper still emits a Node deprecation warning because Windows test command spawning uses `shell: true`; it does not affect correctness, but the helper can be hardened later.
