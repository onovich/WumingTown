# WM-0137 Work report

## Outcome

Implemented the post-M8 camera interaction gate. The Web world now supports mouse/pointer drag panning, wheel zoom, keyboard movement, keyboard reset-to-fit, bounded viewport behavior and localized camera input feedback while preserving click selection. The fix also moves world input handling to a shell-level input target with HUD-control filtering, so empty map/HUD gaps can drive the camera without letting real buttons or settings panels trigger map movement.

## Files and public contracts changed

- `packages/renderer-pixi/src/pixi-world-renderer.ts`: adds camera drag state, bounded pointer pan, reset-to-fit keyboard handling, container/shell input routing and debug center coordinates.
- `packages/renderer-pixi/src/render-geometry.test.ts`: adds pan-bound and reset-to-fit coverage for read-model camera geometry.
- `apps/web/src/shell-bootstrap.ts`: routes shell-frame pointer/wheel input into the renderer while ignoring HUD controls and settings surfaces.
- `apps/web/src/web-shell.e2e.test.ts`: adds camera drag/wheel/keyboard/reset/click-selection E2E coverage and debug payload center fields.
- `packages/ui-react/src/localization.ts`: adds camera drag/reset localized messages.
- `packages/ui-react/src/shell-read-model-localization.ts`: maps camera input labels into localization keys.
- `packages/ui-react/src/shell-hud.test.ts`: covers zh-CN camera input feedback localization.
- `coordination/reports/WM-0137-plan.md`: records the implementation plan.
- `coordination/tasks/WM-0137.json`: task state/history only.

## Acceptance evidence

- Mouse drag pans without breaking click selection: `apps/web/src/web-shell.e2e.test.ts` now verifies pointer drag changes camera center, then reset plus click selection still updates `selectedEntityId`.
- Wheel zoom and keyboard movement remain bounded/deterministic: renderer camera still uses `panViewport` and `zoomViewportAtPoint` from read-model geometry; E2E verifies wheel zoom and ArrowRight movement; `render-geometry.test.ts` verifies pan clamping.
- Reset returns to fitted viewport: keyboard `Home`, `Digit0` and `Numpad0` reset to `createFittedViewport`; E2E compares reset center/zoom to baseline.
- Camera interactions update localized input feedback: HUD labels now localize `Camera drag` and `Camera reset`; E2E sees English feedback and unit coverage verifies zh-CN strings.
- Renderer/unit/E2E coverage exists: focused renderer/UI unit tests and the full Web shell E2E passed.

## Decisions and deviations

- The task-required `corepack pnpm test --filter web-shell` remains a repository command-shape mismatch because `web-shell` is an E2E filter, not a unit filter. This is recorded rather than hidden.
- The fix intentionally listens on the shell frame, not only the canvas, because real hit-testing can land on non-interactive full-screen shell hosts above the canvas. HUD controls are filtered so buttons/settings remain clickable.
- No on-screen reset button was added; the accepted reset affordance is keyboard reset via `Home`, `Digit0` and `Numpad0`.
