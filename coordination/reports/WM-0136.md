# WM-0136 Work report

## Outcome
World selection now behaves as player-facing inspect interaction instead of silent diagnostics-only feedback. Pixi mouse selection updates selected residents and structure-like map objects, empty map clicks create localized tile inspection state, and the React HUD reflects both paths without mutating authoritative world state.

## Files and public contracts changed
- `apps/web/src/product-gate-fixture.ts`: reused an existing support fixture slot as a selectable bridge ledger kiosk structure while preserving the 40 visible actor gate.
- `apps/web/src/shell-bootstrap.ts`: synchronized inspected tile presentation state between Pixi and the shell store.
- `apps/web/src/web-storage-gate.ts`: restored inspected tile state when loading shell evidence from local storage.
- `apps/web/src/web-shell.e2e.test.ts`: added resident, structure and empty-tile selection coverage at windowed and fullscreen-equivalent viewport sizes.
- `packages/renderer-pixi/src/pixi-world-renderer.ts`: added inspected tile state, entity-kind visual markers, tile highlight feedback and selection callbacks.
- `packages/ui-react/src/index.ts`: exported the entity tile helper used by the Web shell.
- `packages/ui-react/src/localization.ts`: added localized empty-tile inspector and terrain labels.
- `packages/ui-react/src/shell-hud.ts`: rendered empty-tile inspection details and terrain feedback when no entity is selected.
- `packages/ui-react/src/shell-hud.test.ts`: covered localized empty-tile inspector rendering.
- `packages/ui-react/src/shell-read-model-localization.ts`: localized the bridge ledger kiosk fixture support names and generated select labels.
- `packages/ui-react/src/shell-store.ts`: added read-only helpers for entity lookup and entity tile lookup.
- `coordination/tasks/WM-0136.json`: recorded the claim/history for this work.

## Acceptance evidence
- Clicking visible residents and map objects updates selected state: `apps/web/src/web-shell.e2e.test.ts` selects `lantern-keeper-shen` and `bridge-ledger-kiosk-04`, then asserts HUD inspector text and `data-selected-entity`.
- Selection feedback is visible without UI-owned world mutation: `packages/renderer-pixi/src/pixi-world-renderer.ts` keeps selection as renderer snapshot state and draws entity/tile feedback from the read model only.
- Clicking empty map tiles gives localized feedback: `packages/ui-react/src/shell-hud.test.ts` asserts `Tile inspection`, localized input text and `Lantern glow`; E2E clicks an empty point and asserts the player inspector changes to tile inspection with terrain.
- Overlay pointer events do not block map selection except over controls: the E2E performs canvas selection with the player HUD active in both windowed and fullscreen-equivalent layouts and reuses the existing layout reachability checks.
- Windowed and fullscreen-equivalent viewport coverage: `node tools/test-runner.mjs e2e --filter web-shell` passed with the new 1424x861 and 2560x1369 selection path.

## Determinism, save, performance and security impact
Determinism: presentation-only selection state is derived from the existing read model and pointer coordinates; it does not change simulation authority, seed streams or worker protocol. Save: the existing Web storage evidence now restores inspected tile presentation state; no public save compatibility claim or schema expansion is introduced. Performance: renderer changes reuse entity iteration already present for draw/hit-test and add only bounded per-pointer/selection work, not per-tick simulation work. Security: no new dependency, network path, Electron privilege or preload surface was added.

## Decisions and deviations
The reviewed fixture target remains 40 visible actors, so the selectable structure path reuses the former support role slot as `bridge-ledger-kiosk-*` instead of adding new notable entities. The task-listed `corepack pnpm test --filter web-shell` command is not supported by this repository's unit runner; `node tools/test-runner.mjs e2e --filter web-shell` was used as the direct web-shell coverage command and passed.

## Remaining risks / follow-up tasks
Camera drag/zoom ergonomics are intentionally left to WM-0137. Wider responsive layout polish remains in WM-0135. This task does not add gameplay command authority; it only makes read-model inspection playable and visible.
