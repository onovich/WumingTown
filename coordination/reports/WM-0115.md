# WM-0115 Work report

## Outcome

WM-0115 closes the localization evidence gap for the current shell chrome by
adding an explicit zh-CN default UI check for the covered player surfaces and
expanding the hardcoded-string inventory so it separates default player UI from
diagnostics-only copy across the web shell diagnostics surfaces. The second
repair pass completes the diagnostics-only web inventory for shell bootstrap
and web storage literals that were still missing in reviewer feedback. The
existing `ui.*` translation coverage remains validated by the localization
gate, and the default Chinese shell path does not expose English player shell
labels for the covered surfaces.

## Files and public contracts changed

- `packages/ui-react/src/shell-hud.test.ts`: adds a zh-CN render assertion for
  the covered shell chrome and confirms English shell labels are absent from
  the default player path.
- `docs/04_content_balance/13_m8_translation_inventory.md`: hardcoded-string
  inventory split into player UI and dev/debug diagnostics, now expanded to
  include the missing diagnostics-only English literals from web diagnostic
  package, shell bootstrap, and web storage surfaces, including the exact
  literals raised in the second review.
- `coordination/reports/WM-0115.md`: task report and verification summary.

## Determinism, save, performance and security impact

None. The change is limited to UI test coverage and documentation. No
simulation authority, save schema, Electron preload scope, runtime storage
behavior or network behavior changed.

## Decisions and deviations

- Followed the M8 locale contract in `docs/05_tech/12_m8_i18n_locale_contract.md`.
- No production localization runtime was changed because the current player
  shell chrome already routes through the complete `zh-CN`/`en` catalog.
- Diagnostics-only hardcoded strings remain in the inventory as allowed by the
  contract because they are gated from the default player launch.

## Remaining risks / follow-up tasks

- Diagnostics surfaces still contain English runtime strings by design. They
  are separated in the inventory, but they are not localized in this task.
- Future player-facing surfaces must continue to use `validateLocalizationCatalogs()`
  and must not introduce new hardcoded shell labels.
