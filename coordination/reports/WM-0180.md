Added a reusable Windows PowerShell 5.1 quiet-host admission entrypoint, a
no-sleep synthetic child-process contract test, and the reviewed workflow
invocation/stop policy. The production entrypoint locks two six-second windows
and the 40/8 percent thresholds, emits one compact JSON stdout line with stable
exit codes, and never invokes a benchmark. WM-0180 performed no live quiet-host
measurement and no benchmark.

## Acceptance evidence

- Windows PowerShell 5.1 is the mandatory runtime and passed the explicit major
  version check. Both scripts parse under the Windows PowerShell parser.
- Finite validation uses `IsNaN` plus `IsInfinity`; neither script references
  `Double.IsFinite`, `Get-Counter`, CIM, WMI, process affinity, priority, or a
  benchmark command. The self-test contains no `Start-Sleep`; the production
  script contains the only fixed six-second sleep.
- The production script accepts only repository root, exact full commit, and
  the explicit no-concurrent-benchmark confirmation. Window count, duration,
  and thresholds are constants rather than caller options.
- The 30-case child matrix covers finite `0`, negative zero, `24.41`, exact
  `40/8` boundaries and `Double.MaxValue`; null, non-numeric, NaN and both
  infinities; array, duplicate, missing and wrong-count shapes; zero,
  fractional and Int32-overflow logical CPU counts;
  dirty/commit/ACShadows/confirmation preconditions; threshold
  failures; postconditions; invalid invocation; and caught unexpected errors.
- Every child case asserted one parseable compact JSON stdout line, empty
  stderr, schema/evidence/status/reason fields, locked constants, and exact
  process/JSON exit agreement for codes `0`, `2`, `10`, `20`, `30`, `40`, and
  `70`. Direct raw-stdout shape checks reject missing terminators, extra blank
  lines and duplicate JSON lines before trimming or parsing.
- A measured-path probe on the intentionally dirty implementation tree stopped
  before sampling with one JSON line, `reasonCode=git_dirty_before`, and JSON
  exit code `10`. It did not sleep or measure.
- All three changed workflow files are UTF-8 without BOM. The production and
  self-test scripts are 597 and 325 lines respectively. The production file is
  above the general 400-line review threshold because WM-0180 explicitly owns
  only one production script and one self-test script; splitting a fourth
  helper file is forbidden by the reviewed three-file scope.

## Decisions and deviations

The implementation follows the WM-0180 architecture packet. Three delegated QA
turns were interrupted after leaving the claimed branch clean and writing no
files; the project-director then acted as the `qa-performance` implementer to
avoid an orchestration stall. Final review remains assigned to a separate
reviewer, preserving the implementer/reviewer separation. No scope or gate
deviation was made. The first independent review requested two medium fixes:
strict positive-Int32 logical CPU validation and raw stdout line-count
validation. Both were added with fractional/overflow and extra-line regression
coverage before re-review.
