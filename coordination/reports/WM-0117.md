# WM-0117 Work report

## Outcome

The changes-requested repair is complete. The post-start town view still
defaults to the player-facing HUD with diagnostics gated behind
`wmDiagnostics=1`, but the layout is now viewport-bounded: desktop mode uses
scrollable left/right HUD columns inside the available viewport, and compact
mode collapses into a single reachable scroll stack with a bounded selected
detail card instead of overlapping absolute rails.

## Files and public contracts changed

- `packages/ui-react/src/shell-hud.ts`: replaced the absolute post-start rails
  with bounded desktop columns and a compact stacked layout; kept the gated
  debug overlay path intact.
- `packages/ui-react/src/localization.ts`: added localized player HUD labels,
  phase meanings, night-risk tiers, entity kinds, and need-state strings.
- `packages/ui-react/src/shell-settings-panel.ts`: fixed compact-width sizing so
  the settings panel no longer forces horizontal overflow in the HUD top rail.
- `packages/ui-react/src/shell-hud.test.ts`: added a town-view/debug-mode
  render assertion alongside the default main-menu assertion.
- `apps/web/src/product-gate-fixture.ts`: rewrote visible default HUD copy to
  read as player-facing town state instead of product-gate harness evidence.
- `apps/web/src/web-shell.e2e.test.ts`: added explicit player-HUD and
  debug-overlay assertions plus a focused viewport test for `1424x861` and
  `390x720` that checks required-panel reachability and no cover/overlap at the
  point of interaction.
- `apps/desktop-electron/src/desktop-shell.e2e.test.ts`: added explicit player
  HUD assertions for default desktop flow and debug-overlay assertions for the
  `wmDiagnostics=1` path.
- `coordination/reports/WM-0117-plan.md`: execution plan used before edits.
- `coordination/tasks/WM-0117.json`: task state/history updated by workflow
  claim command.

## Determinism, save, performance and security impact

- Determinism: none. No simulation or Worker logic changed.
- Save: none. No save format, storage schema, or import/export protocol changes.
- Performance: low-risk UI-only change. The HUD still consumes existing
  read-model state; the repair replaced overlap-prone absolute rails with
  bounded scroll regions and passed focused viewport reachability checks.
- Security: none. Electron sandbox/preload boundaries were not widened; debug
  evidence remains behind the explicit diagnostics path.

## Decisions and deviations

- Stayed within allowed paths; no Worker protocol or sim package edits.
- Kept diagnostics English-only inside the explicit debug overlay, which is
  consistent with the M8 scope amendment and design-system allowance for
  developer diagnostics outside the default player UI.
- Adjusted the existing product-gate fixture's default visible copy so the
  post-start HUD reads like player-facing state instead of reviewed evidence.
