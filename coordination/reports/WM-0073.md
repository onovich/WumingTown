# WM-0073 Work report

## Outcome

Implemented the M5 data-only content validation gate for alpha content packs. The gate validates M5 manifests, paths, file sizes, JSON/JSON5 syntax, schema refs, M5 semantics, localization coverage and unsupported capabilities, then compiles accepted packs into deterministic DefIndex order with a stable content manifest hash and validation counters. Reviewer-requested repairs now enforce manifest-declared `maxFileBytes` and `maxTotalBytes` against actual pack bytes after manifest parse, and the public validator measures bytes from `file.text` instead of trusting caller-provided `byteLength` metadata. No runtime M5 behavior, Worker protocol, save format, package manifest or lockfile changes were made.

## Determinism, save, performance and security impact

- Determinism: M5 pack directory loading, manifest arrays, DefIndex and hash inputs use stable sorting. Tests verify hash stability under input file order changes. Byte counters are derived from stable UTF-8 text measurement at validation time.
- Save/replay: compiled output carries a stable content manifest hash and counters suitable for later save/replay basis checks, but WM-0073 does not change save format or Worker protocol.
- Performance: validation is pre-runtime content tooling work only. No simulation hot path, runtime benchmark baseline or package dependency changed.
- Security: data mods fail closed for unsafe paths, code/executable/script files, archives, remote URL dependencies, unsupported capabilities, oversized files, manifest-declared size cap violations, spoofed `byteLength` metadata and invalid schema/semantics/localization.
