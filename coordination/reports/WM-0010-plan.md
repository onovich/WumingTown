# WM-0010 Plan

## Objective

Establish reproducible local and CI quality gates for deterministic simulation diagnostics, benchmark baseline comparison, and dependency auditing without introducing silent retries or non-reproducible pass conditions.

## Current Gaps

- `.github/workflows/ci.yml` runs only a single Windows job and does not record exact tool or environment versions.
- Root scripts do not provide the task-required `pnpm ci:local` or `pnpm audit:deps` entry points.
- `pnpm bench` emits one benchmark report but does not compare against an explicit baseline or fail on explained thresholds.
- Replay diagnostics expose first divergence details in-memory, but current CLI output does not package seed, divergent tick, and artifact paths for long-run or CI failure triage.
- E2E tests poll for readiness, but there is no explicit guardrail ensuring the repo does not use framework-level silent retries for nondeterministic behavior.

## Planned Changes

1. Add explicit root scripts for local CI reproduction, dependency audit, deterministic replay diagnostics, and environment/version capture.
2. Extend the benchmark package so the CLI can run named suites, write machine-readable artifacts, load a checked-in baseline, compare deltas, and enforce documented regression thresholds.
3. Add a long-run diagnostics script that writes deterministic replay artifacts and prints structured failure details including seed, first divergent tick, and artifact paths.
4. Upgrade GitHub Actions to run on pull requests, capture exact Node/pnpm/OS metadata, run the local CI scripts, and upload benchmark/diagnostic artifacts.
5. Audit test configuration so failures surface immediately and no workflow or test runner silently retries nondeterministic product behavior.
6. Update project docs/reporting as needed so reviewers can reproduce the same commands locally.
