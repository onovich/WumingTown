# WM-0124 Work Report

## Outcome

Produced an M8 data-mod workflow and localization completeness report without
touching forbidden runtime surfaces. The task now has an explicit M8
content/public-text inventory, and the content schema test suite includes a
focused M5 workflow regression for missing references plus required-locale
and locale-file probes so the fail-closed path is exercised alongside the
existing unsafe-path, oversized-asset and missing localization coverage. The
reviewer probes for `manifest.locales=["en"]` and
`manifest.locales=["en","zh-CN"]` with only `locales/zh.json` now fail closed
because M5/M8 packs must declare English plus a Chinese locale resource;
`zh-CN` is canonical and must resolve to a canonical `zh-CN` file when it is
declared, and legacy `zh` compatibility only applies when a pack explicitly
declares `zh`.

## Files changed

- `packages/content-schema/src/m5-content-validation.test.ts`: added a focused
  M5 workflow regression that rejects missing references before runtime
  consumption and added required-locale regression probes for English plus a
  Chinese locale resource, including the `zh.json` bypass probe for declared
  `zh-CN`.
- `packages/content-schema/src/m5-content-manifest.ts`: added fail-closed
  required-locale validation for English plus a Chinese locale resource, with
  `zh-CN` requiring an actual `locales/zh-CN.json` or `.json5` file and
  legacy `zh` accepted only when a pack explicitly declares `zh`.
- `docs/04_content_balance/13_m8_translation_inventory.md`: expanded with an
  M8 content/public-text inventory, the required `en`/`zh-CN` content text
  families, the fail-closed workflow notes, and the required-locale gate.
- `coordination/reports/WM-0124.md`: this report.
- `coordination/tasks/WM-0124.json`: taskctl state updated by claim and
  completion workflow.
