# WM-0035 Work report

## Outcome

Bound indexed WorkOffer selection to versioned local A\* pathing for the M2
path/work-selection slice. The sim-core helper consumes only caller-supplied
indexed candidates, prefilters stale region rows and blocked targets, exact
paths bounded Top-K candidates, and rejects stale path bases before exposing a
selectable work path. No job, reservation, item, save, Worker or UI state is
mutated by this task.

## Files and public contracts changed

- `packages/sim-core/src/work-offers.ts`: added internal
  `selectPathResolvedWorkOffer` helper, M2 path-selection options/scratch/result
  types, and structured reason counters.
- `packages/sim-core/src/work-offers.test.ts`: added M2 path-selection tests for
  indexed candidate bounds, deterministic ordering, stale basis rejection,
  region-unreachable, blocked and node-budget reasons.
- `packages/sim-core/src/pathing.test.ts`: added focused blocked, no-route and
  node-budget path reason coverage.
- `packages/benchmarks/src/m2-path-work-selection-benchmark.ts`: added focused
  WM-0035 benchmark and artifact writer.
- `packages/benchmarks/src/index.ts`: exported the focused benchmark helper.
- `docs/02_systems/03_map_space_rooms_lanterns.md`: documented WM-0035 region
  and stale-basis behavior.
- `docs/02_systems/04_ai_jobs_reservations_pathing.md`: documented the helper
  contract, caps and reason classes.
- `docs/05_tech/03_performance_budget.md`: documented the focused benchmark and
  CLI-filter substitution.

No public Worker protocol, save format, schema, dependency, package manifest or
UI-owned simulation contract changed.

## Acceptance evidence

- Indexed candidates only: `selectPathResolvedWorkOffer` calls
  `WorkOfferIndex.selectTopOffers` with caller-provided caps/scratch and reads
  only the selected offer rows; tests prove off-bucket offers are not selected.
- Version bases and stale rejection: path requests carry `PathVersionBasis`;
  `work-offers.test.ts` verifies stale basis returns
  `work_path_version_basis_stale` before a selected path is exposed.
- Bounded Top-K exact pathing: the helper exact paths at most `maxExactPaths`;
  tests and benchmark record exact-path cap hits.
- Structured reasons: pathing tests cover blocked/no-route/node-budget; work
  selection tests cover region-unreachable, stale, blocked and node-budget
  counters.
- Artifact-backed metrics: direct WM-0035 benchmark artifact at
  `coordination/artifacts/WM-0035/benchmarks/m2-path-work-selection-results.json`
  reports 100 exact path requests, 25 stale rejects, queue backlog peak 1,
  final queue backlog 0, and 5,500 A\* node expansions. This generated
  artifact is intentionally retained for review because acceptance criterion 5
  requires artifact-backed queue, node and stale-reject metrics, even though the
  task allowedPaths did not explicitly include
  `coordination/artifacts/WM-0035/**`.

## Determinism, save, performance and security impact

Determinism: selection order remains score descending with offer id tie-breaks;
all caps and counters are integer lanes. Path basis comparison is explicit and
rejects stale results before job/reservation mutation.

Save/replay: no save format, schema or persisted cache changed. The helper owns
no long-lived actor path and writes no authoritative state.

Performance: normal work selection is bounded by caller caps and indexed bucket
iteration. The benchmark records `candidateCap = 24`, `selectedCap = 12`,
`exactPathCap = 4`, candidate cap hits, exact-path cap hits, queue backlog and
node expansions. `selectPathResolvedWorkOffer` is over the preferred function
length; it is kept in one pass to avoid duplicate candidate/path counter walks,
and should be split if later tasks add reservation/job mutation.

Security and boundaries: `sim-core` remains free of DOM, React, Pixi, Electron,
Node filesystem, real time and ambient randomness. The new Node filesystem use
is limited to the benchmark artifact writer in `packages/benchmarks`.

## Decisions and deviations

ADR-0007 is followed: WorkOffer rows remain derived, path results are basis
data, and no public protocol/save/schema/package-boundary change was introduced.

Required filters `m2-path-work-selection` are not supported by the existing
test/benchmark CLIs, and the registry files needed to add them are not in
WM-0035's allowed paths. The substitutions above are therefore explicit and
limited to existing focused filters plus the exported benchmark artifact helper.

The generated WM-0035 benchmark artifact is outside the explicit allowedPaths
list, but it is required evidence for the task's artifact-backed metrics
acceptance. The unrelated generated WM-0022 `pathing-100` artifact side effect
from the required benchmark command was restored and is not part of the final
intended diff.

## Remaining risks / follow-up tasks

- Future integration into job reservation/commit code must call the helper
  before mutation and must not treat the returned path as persisted owner state.
- Benchmark CLI support for `m2-path-work-selection` should be added by a task
  whose allowed paths include the benchmark registry and baseline policy files.
- Reservation conflict, material availability and final job mutation reasons
  remain for WM-0037/WM-0038/WM-0039.
