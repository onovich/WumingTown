## Decisions and deviations

- I added `coordination/artifacts/WM-0095/**` to WM-0095 allowed paths because
  the task explicitly needs consolidation artifacts and the original task JSON
  did not include an artifact output path.
- I ran `pnpm bench` with `WM_ARTIFACT_DIR=coordination/artifacts/WM-0095` to
  preserve the required script entry point while avoiding any rewrite of the
  verified WM-0083 benchmark artifact.
- I reran `tools/web-performance-gate.mjs` after `pnpm test:e2e` because e2e
  rebuilds the Web renderer, and the Web performance report should match the
  final release-gate report in `apps/desktop-electron/dist/renderer`.
- First independent review requested a blocking fix because the consolidation
  script claimed benchmark-threshold protection without explicitly asserting the
  `10` percent warning and `20` percent failure thresholds or the WM-0095
  artifact root. I fixed the script to fail unless every benchmark comparison
  has thresholds exactly `10` / `20`, `baselinePath` is
  `packages/benchmarks/baseline.json` and `artifactPath` is
  `coordination/artifacts/WM-0095/benchmarks/benchmark-results.json`, then
  regenerated the consolidation JSON and SHA-256 sidecar.
