## Determinism, save, performance and security impact

Determinism: no runtime code changed. The contract explicitly keeps localized
text out of Simulation Worker/headless authority and requires structured keys,
reason codes and parameters instead of localized simulation prose.

Save impact: no save schema, save migration or save compatibility promise. The
contract states locale preference is app-shell preference storage only and not
Save Container state.

Performance impact: no benchmark thresholds changed. Future localization work
is constrained to app-shell dictionary selection and UI render lookup, with
completeness and pseudo-localization checks as build/test work, not simulation
hot paths.

Security impact: no Electron code changed. The contract rejects broad Electron
APIs and allows only a future narrow typed locale/preference bridge if packaged
evidence requires it and the preload audit is updated.

## Decisions and deviations

Added ADR-0011 in addition to the focused tech doc because WM-0113 is an
architecture contract task and the decision needs recorded alternatives,
migration, rollback, test and performance implications. This is within
WM-0113 allowed paths.

and is explicitly not Spark-eligible under the project lane rules.

runtime pnpm wrapper was version `11.7.0`. The required gate passed after using
