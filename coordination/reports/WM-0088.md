# WM-0088 Work report

## Outcome

Implemented a Web OPFS storage gate for the WM-0086 product shell: the browser
now saves a narrow M6 gate evidence envelope with A/B slot writes, can load it
back into the read-only shell, exports it as a browser download, imports it
from a file input, and surfaces structured quota and envelope failures without
leaking local paths, secrets or full-save contents. Windows/Web container
interoperability remains blocked because the current Electron preload still
returns placeholder unavailable save ports, so WM-0088 records that blocker
instead of over-claiming cross-platform compatibility.

## Files and public contracts changed

- `packages/persistence/src/save-store-types.ts`: typed save-store result,
  quota, export and OPFS adapter contracts for the M6 gate surface.
- `packages/persistence/src/m6-gate-save.ts`: versioned M6 gate envelope codec
  and SHA-256 helper; explicitly scoped to gate evidence, not public save
  compatibility.
- `packages/persistence/src/opfs-save-store.ts`: OPFS A/B slot save store with
  manifest pointer, quota preflight, integrity checks and structured failures.
- `packages/persistence/src/opfs-save-store.test.ts`: focused persistence tests
  for write/list/export/read/remove, quota failure preservation and invalid ids.
- `packages/persistence/src/index.ts`: public persistence exports for the new
  store and gate envelope helpers.
- `packages/platform/src/index.ts`: Web fallback now resolves to the OPFS save
  store, exports gate-envelope helpers for the Web shell, and surfaces optional
  storage status/export operations while keeping desktop placeholder bridges
  valid.
- `packages/ui-react/src/shell-store.ts`: shell state now carries storage gate
  status, diagnostics, quota and action contracts.
- `packages/ui-react/src/shell-storage-panel.ts`: user-facing storage gate UI
  for save/load/export/import/delete/refresh plus structured diagnostic output.
- `packages/ui-react/src/shell-hud.ts`, `shell-hud.test.ts`, `index.ts`:
  mounts and verifies the storage gate panel in the Web HUD.
- `apps/web/src/web-storage-gate.ts`: Web shell controller that persists only
  read-only gate envelope state, restores selection on load, triggers browser
  downloads and records the Windows interoperability blocker.
- `apps/web/src/shell-bootstrap.ts`: wires the storage controller into the
  shell state and debug payload without changing simulation authority.
- `apps/web/src/web-shell.e2e.test.ts`: Playwright coverage for OPFS
  round-trip, export/import and quota-failure recovery.
- `tools/test-runner.mjs`: project-director-approved narrow exception to add
  only the `persistence` filter route required by `pnpm test --filter
  persistence`; it does not change other filters or test semantics.
- `docs/05_tech/04_persistence_mods_security.md`: WM-0088 note for OPFS
  A/B-slot writes, gate-envelope scope and structured failure constraints.
- `docs/05_tech/06_platform_matrix.md`: records the proven Web gate path and
  the current Windows interoperability blocker.
- `coordination/reports/WM-0088-plan.md`: required execution plan.
- `coordination/tasks/WM-0088.json`: project-director narrow-scope exception

## Acceptance evidence

1. OPFS save, load, export, import and quota-failure recovery are tested.
   - `packages/persistence/src/opfs-save-store.test.ts` covers A/B-slot
     write/list/export/read/remove, quota rejection preserving the old slot,

   - `apps/web/src/web-shell.e2e.test.ts` covers:
     - save fixture -> export download -> delete -> import file -> load;

     `data-testid="storage-status"` and structured diagnostics at
     `data-testid="storage-diagnostic"`.

2. Windows/Web save container interoperability is proven or recorded as a blocker.
   - Web side is proven through the OPFS and Playwright evidence above.
   - Windows side is blocked as of WM-0088:
     `apps/desktop-electron/src/preload.cjs` still exposes placeholder
     unavailable save-store ports only, so no reviewed Windows reader/writer
     can yet consume the same container.
   - The blocker is surfaced in the UI storage panel and documented in
     `docs/05_tech/06_platform_matrix.md`.

3. Error states produce user-facing and diagnostic structured reasons.
   - Persistence returns typed `SaveStoreError` records with code, diagnostic
     detail, message, recoverable flag and user message.
   - Platform preserves those structured reasons for the Web shell.
   - UI renders user-facing text plus a diagnostic JSON block; Playwright
     asserts the quota failure shows `quota_exceeded`.

4. No public save compatibility promise is made beyond the M6 gate.
   - `packages/persistence/src/m6-gate-save.ts` hard-codes the authoring note
     that the payload is an M6 gate evidence envelope only.
   - `docs/05_tech/04_persistence_mods_security.md` documents that WM-0088 is
     not a public compatibility, schema or codec commitment.
   - No `packages/sim-core/**` or save codec/migration files changed.

## Determinism, save, performance and security impact

- Determinism: none. WM-0088 does not touch authoritative simulation or Worker
  logic; it persists read-only shell evidence only.
- Save: adds only an M6 gate envelope and OPFS/browser plumbing. It does not
  create public save compatibility, codec migration, schema migration or
  `sim-core` save authority changes.
- Performance: normal hot paths are unchanged. The new storage logic runs only
  on explicit save/load/export/import actions and quota/status refresh.
- Security: structured diagnostics are limited to ids, byte sizes, checksums
  and reason codes. No local paths, secrets or full-save contents are emitted.

## Decisions and deviations

- Project-director narrow-scope exception: `tools/test-runner.mjs` was added to
  `allowedPaths` only so `pnpm test --filter persistence` routes to the new
  persistence test file. No other tool files or test-runner behaviors were
  changed.
- The Web shell stores only a gate-scoped read-only envelope because the
  current browser path is still a deterministic product shell, not an
  authoritative simulation runtime. WM-0088 therefore proves storage/import/
  export plumbing without promising public gameplay save compatibility.
- Windows interoperability is explicitly reported as blocked rather than hidden
  behind the Web-only pass.

## Remaining risks / follow-up tasks

- A future Windows/Electron task must replace the placeholder preload save
  ports with a reviewed minimal bridge before M6 can claim Windows/Web
  container interoperability.
- The current gate envelope restores shell selection and metadata only; it is
  intentionally narrower than future authoritative gameplay save scope.
- Browser quota estimates can still vary by engine, so quota preflight remains
  conservative even though the Web recovery path is now observable and tested.
