# WM-0119 Work report

## Outcome

Added M8 first-play guidance to the default player start surface. The UI now
explains phase meaning, available shell actions, the next visible goal and the
diagnostics boundary in zh-CN/en before the player enters the town HUD. Review
repair removed developer-gate terminology from default player copy and added
localized zh-CN display copy for phase and next-goal values derived from the
read model.

## Files and public contracts changed

- `packages/ui-react/src/shell-main-menu-surface.ts`: adds the default
  first-play guidance cards, reviewer evidence test ids, and localized zh-CN
  display derivation for phase and next-goal copy.
- `packages/ui-react/src/shell-hud.ts`: passes visible next-goal and phase
  meaning data from the read model into the start surface.
- `packages/ui-react/src/localization.ts`: adds zh-CN/en player-facing guidance
  strings and removes developer-gate terminology from default player boundary
  copy.
- `packages/ui-react/src/shell-hud.test.ts`: verifies default guidance,
  bilingual shell copy and diagnostics separation.
- `apps/web/src/web-shell.e2e.test.ts`: extends Web start-surface evidence for
  first-play guidance.
- `apps/desktop-electron/src/desktop-shell.e2e.test.ts`: extends desktop
  start-surface evidence for first-play guidance.
- `docs/01_design/10_m8_first_play_guidance.md`: records the M8 guidance
  contract, copy boundaries, cultural terminology boundaries and provisional
  values.
- `coordination/reports/WM-0119-plan.md`: records the execution plan created
  before implementation.

## Acceptance evidence

- Default UI explains current phase, available actions and next goal in
  zh-CN/en: `shell-main-menu-surface.ts` renders localized phase guidance,
  `main-menu-available-actions` and `main-menu-next-goal`; `shell-hud.test.ts`
  and Web/Desktop e2e baselines assert these surfaces in both locales. Repair
  evidence asserts zh-CN shows `黄昏守望`, `补上灯火缺口` and the localized
  lamp-duty detail instead of raw English fixture strings.

  player guidance is present while developer-gate wording is absent from the
  start surface; diagnostics remain gated behind `wmDiagnostics=1`.
- Extends M7 onboarding into M8 first-play guidance without release claims:
  `docs/01_design/10_m8_first_play_guidance.md` records controlled-test and
  demo-only copy boundaries.
- Records copy limits and cultural terminology boundaries:
  `docs/01_design/10_m8_first_play_guidance.md` records blocked claims and
  canonical terminology boundaries.
- Covered by tests or reviewer-verifiable UI evidence: `main-menu-*` test ids
  and updated unit/e2e assertions provide reviewer-visible evidence.
