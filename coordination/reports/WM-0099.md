# WM-0099 Work report

## Outcome

WM-0099 adds a read-model-only M7 first-run onboarding path to the existing
Web/Windows shell HUD and verifies it through Web and desktop Electron smoke
coverage. The path explains the current M5/M6 shell surfaces for external
players without changing simulation authority, save schema, Worker protocol,
release verdicts, public feedback, telemetry, account, paid service or M8
content scope.

## Files and public contracts changed

- `packages/ui-react/src/shell-store.ts`: adds typed onboarding state to the
  shell store contract.
- `packages/ui-react/src/shell-onboarding-panel.ts`: renders the M7 first-run
  onboarding panel with explicit authority and release boundaries.
- `packages/ui-react/src/shell-hud.ts`: places onboarding inside the existing
  product-gate HUD card and constrains that card with internal scrolling.
- `packages/ui-react/src/index.ts`: exports onboarding panel/types through the
  public package entry.
- `packages/ui-react/src/shell-hud.test.ts`: covers server-rendered onboarding
  output in the UI package smoke test.
- `apps/web/src/shell-bootstrap.ts`: initializes the M7 first-run path copy for
  the Web shell.
- `apps/web/src/web-shell.e2e.test.ts`: asserts onboarding copy, step count and
  boundaries on the Web shell surface.
- `apps/desktop-electron/src/desktop-shell.e2e.test.ts`: asserts the same
  onboarding path on Electron dev-server and Windows packaged-directory smoke.
- `docs/01_design/08_m7_first_run_onboarding.md`: records scope, authority
  boundary, copy limits and unresolved risks.
- `coordination/reports/WM-0099-plan.md`: records the execution plan and
  missing input paths.
- `coordination/tasks/WM-0099.json`: taskctl claim/complete history only.

No `sim-core`, `sim-worker`, Worker protocol, save schema, runtime dependency,
telemetry, account, paid service, crash upload, public feedback, release,
store, signing, installer, updater or M8 file is changed.

## Determinism, save, performance and security impact

Determinism impact: none. No simulation, PRNG, tick runner, scenario,
benchmark baseline or authority code changes.

Save impact: no save schema, migration or public compatibility promise changes.
The onboarding copy points to existing Web OPFS save/import/export evidence and
keeps Windows host save bridging blocked.

store submission or paid service.

## Decisions and deviations

- Roadmap authority remains current repository Roadmap: M6 = Web / Windows
  Product Gate, M7 = Early Access / public playtest preparation, M8 = 1.0.
- Web remains exactly `demo-only`.
- Windows remains unsigned `ready-for-controlled-external-test`.
- The onboarding panel is first-run guidance for existing surfaces, not a
  command surface and not simulation authority.
- Required task inputs `docs/01_design/00_product_brief.md` and
  `docs/01_design/03_player_experience.md` are absent from the repository.
  The implementation used existing design overview, player journey, daily loop
  and UI/UX documents instead and records this deviation in the plan/design
  note.
- Initial `pnpm quality` failed only on Prettier formatting for
  `packages/ui-react/src/index.ts` and
  `packages/ui-react/src/shell-onboarding-panel.ts`; project Prettier was run
  on those files and the full quality gate then passed.

## Copy limits and unresolved risks

Player-facing copy limits:

- Explains existing M5/M6 shell surfaces only; no M8 tutorial volume.
- Web remains `demo-only`; no same-spec, lower-fast-forward, lower-cap or
  public-release claim.
- Windows remains unsigned controlled external test; no signing, installer,
  updater, store upload or public release.
- No telemetry, accounts, paid service, crash upload or public feedback flow.
- Public save compatibility remains draft/owner-gated.

Unresolved risks:

- The shell fixture can orient testers but is not final browser authority
  runtime evidence.
- Windows host save and diagnostic package writing remain blocked until a
  reviewed narrow desktop bridge exists.
- Later M7 tasks still need tester protocol, known issues, store/playtest
  material drafts and readiness matrix before M7 closeout.
