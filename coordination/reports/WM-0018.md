## Determinism, save, performance and security impact

Determinism: documentation improves M1 determinism guidance by proposing
integer/fixed-point scales, phase ordering, total command ordering, stable
candidate ordering, named random stream use, and versioned async stale rejection.
No deterministic runtime behavior was changed.

Save: no save format implementation changed. ADR-0006 proposes future scale and
contract version migration expectations and aligns with ADR-0005 Save Container
v1 sections.

Performance: no benchmark baseline or product code changed. ADR-0006 proposes
where sorting is allowed, how Top-K caps must be documented, which counters
future benchmarks must report, and that deterministic measurements should occur
at phase boundaries.

Security: no runtime dependencies, filesystem, IPC, network, Worker authority,
Electron, or UI mutation surface changed. ADR-0006 keeps external data and async
results behind validation and structured stale rejection.
