# WM-0123 Work report

## Outcome
Implemented a headless-only faction/endgame owner slice for M8. The new
`M8FactionEndgameStore` records six required faction arcs as decomposed,
versioned owner facts and evaluates the five endgame routes through bounded
route lanes with explicit `seeded -> available|blocked|contested` state
transitions and structured reasons. The protected M5 alpha scenario and hashes
remain unchanged.

Changes-requested repair: duplicate active arc/route registrations are now
rejected before owner-version, index or metric mutation, and all reviewer-named
masks are validated as safe integer masks inside approved domain bits before
typed-array storage.

## Files and public contracts changed
- `packages/sim-core/src/m8-faction-endgame-types.ts`: numeric faction ids,
  endgame route ids, masks, serializable state constants, structured reasons,
  duplicate-registration reasons, query shapes and metrics.
- `packages/sim-core/src/m8-faction-endgame-store.ts`: typed-array owner store
  with faction/route lanes, owner-version checks, duplicate-registration
  rejection, arithmetic safe-integer mask validation, cap-aware queries and
  route state transitions.
- `packages/sim-core/src/m8-faction-endgame-scenario.ts`: deterministic
  headless scenario evidence for six faction arcs and five endgame routes,
  including replay hash and performance counters.
- `packages/sim-core/src/m8-faction-endgame.test.ts`: focused owner-store and
  scenario coverage for decomposed facts, route states, structured reasons,
  replay equality and bounded visits.
- `packages/sim-core/src/m5-invariants.test.ts`: invariant hook confirming the
  M8 endgame scenario remains replay-stable and does not touch protected M5
  baselines; protected M5 expected hashes were not changed.
- `packages/sim-core/src/index.ts`: public sim-core exports for the new owner
  store, constants, types and headless scenario.
- `docs/02_systems/09_obligations_pacts_factions.md`: WM-0123 owner-store and
  route-state implementation note.
- `docs/04_content_balance/11_m8_1_0_content_endgame_scope.md`: WM-0123
  implementation note mapping the M8 contract to the new headless evidence.
- `coordination/tasks/WM-0123.json`: taskctl claim state.
- `coordination/reports/WM-0123.md`: this report.

## Acceptance evidence
- Simulation Worker/headless authority only: all implementation is in
  `packages/sim-core`; no `apps/**`, `packages/ui-react/**`,
  `packages/renderer-pixi/**`, platform, telemetry, account, paid-service or
  release paths changed.
- Bounded indexes/state machines/reasons: `M8FactionEndgameStore` uses typed
  arrays, faction lanes and route lanes; faction arcs transition
  `seeded -> negotiated|failed`; routes transition
  `seeded -> available|blocked|contested`; route results include structured
  reasons such as `m8_faction_endgame_route_available`,
  `m8_faction_endgame_route_blocked_low_support` and
  `m8_faction_endgame_route_contested`.
- Changes-requested repair probes: `m8-faction-endgame.test.ts` now reproduces
  duplicate active arc and route registration after state mutation and asserts
  state/index/counts remain unchanged. It also rejects negative, zero where
  required, fractional, out-of-domain and 32-bit-wrap-sized masks for arc
  contradiction/negotiation/failure masks and route faction support/opposition
  masks before storage.
- Headless scenario and invariants: `runM8FactionEndgameScenario()` returns
  scenario hash `0x5ca30054`, replay hash equality
  `0x5ca30054 == 0x5ca30054`, 6 negotiated faction arcs, and route outcomes:
  2 available, 1 blocked, 2 contested. `m5-invariants` now asserts this
  evidence while preserving protected M5 hashes.
- Protected M5 baseline: `corepack pnpm sim:run -- --seed 5 --scenario
  m5-alpha-content-framework --ticks 100000` reported content
  `0xe55d3015`, command `0x81d37435`, final world `0xfba70a5c`, read-model
  `0x9ba83cb7`.
- Performance measurements: WM-0123 scenario records `factionFactVisits: 42`,
  `routeVisits: 5`, `arcVisits: 6`, `capHitCount: 0`, `staleRejectCount: 0`,
  `maxRouteCandidateCap: 1`, `maxRouteScanCap: 1`.

## Determinism, save, performance and security impact
Determinism: new scenario is deterministic and records replay hash equality.
Save: no public save container, schema, migration or Worker protocol changes.
Performance: route and arc reads are bounded by caller caps and scenario
metrics record visits/cap hits. Security: no new dependencies, no Node
filesystem, DOM, React, Pixi, Electron, real time, ambient randomness, network,
telemetry, account, paid-service or release behavior.

## Decisions and deviations
- ADR-0010 and the M8 scope contract are followed: owner stores remain
  authoritative; derived/UI/prose state is not authority.
- Full shipped faction/endgame campaign content remains broader than this
  vertical slice. This task implements the smallest verifiable authoritative
  integration surface and scenario evidence without touching protected M5
  hashes.
- Spark lane not used: WM-0123 is broad deterministic simulation owner work,
  not a small low-risk Spark patch.
