# WM-0132 Work report

## Outcome

WM-0132 keeps the default Web shell path player-facing while preserving
diagnostics behind explicit debug access only. The main menu and player HUD no
longer present fixture-facing wording as primary content, and the explicit
`wmDiagnostics=1` route now labels the dark technical overlay as developer
diagnostics rather than part of the player HUD.

## Files and public contracts changed

- `packages/ui-react/src/localization.ts`: removed remaining player-visible
  fixture-facing start-surface copy and replaced it with player-facing wording
  in both `en` and `zh-CN`.
- `packages/ui-react/src/shell-hud.ts`: relabeled the explicit debug surface as
  developer diagnostics and kept the separation message inside the debug-only
  overlay.
- `packages/ui-react/src/shell-hud.test.ts`: added assertions that the default
  route does not leak fixture/product-gate identifiers and that the explicit
  diagnostics route exposes the relabeled debug overlay.
- `apps/web/src/web-shell.e2e.test.ts`: added route-level assertions that the
  default start surface and default player HUD stay free of fixture/product-gate
  copy while the explicit diagnostics route exposes the debug-only overlay.
- `coordination/reports/WM-0132-plan.md`: execution plan written before product
  edits.
- `coordination/tasks/WM-0132.json`: task claim/completion workflow state only.

No Simulation Worker authority, save schema, public protocol, Electron
security boundary or platform verdict contract changed.

## Decisions and deviations

- Kept the explicit diagnostics route at `wmDiagnostics=1` rather than adding a
  new toggle path, because the existing explicit route already satisfies the
  separation requirement and keeps the scope within allowed files.
- Updated only player-visible default-route copy and explicit debug labeling;
  no fixture/read-model authority plumbing was changed.
- `corepack pnpm test --filter web-shell` remains unsupported in this repo. The
  required command was still run and recorded exactly, and the matching
  `web-shell` E2E suite passed.
- `node tools/test-runner.mjs e2e --filter web-shell` refreshed
  `coordination/artifacts/WM-0118/**` during verification. Those out-of-scope
  files were restored before final diff review.

## Remaining risks / follow-up tasks

- The default route is now player-facing, but the runtime still depends on the
  controlled fixture/read-model slice; future tasks can replace more of that
  provisional content with fully structured player data.
- Debug overlay strings remain English-only by design because they stay behind
  explicit diagnostics mode. If diagnostics access expands later, those strings
  should move under the debug-localization policy explicitly.
- WM-0138 still owns real command-chain authority. WM-0132 keeps the action bar
  as read-model-facing placeholders only.
