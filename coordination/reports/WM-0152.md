## Determinism, save, performance and security impact

- Determinism: Web only uses public Worker session helpers and
  `drainWebPlayableCommandsToTerminal` with fixed tick steps; it does not
  parse internal advance command ids or create a local runtime.
- Save: no public save compatibility is promised. In-progress and completed
  authoritative command runtime state is explicitly not restored from the Web
  OPFS gate envelope.
- Performance: Pixi/React remain read-model consumers. Projection updates are
  batched through the Worker helper rather than a browser hot-poll advance loop,
  and no per-cell display-object or per-sprite ticker path was added.
- Security/boundaries: no `sim-core` import, no sim-worker subpath import, no
  package-boundary relaxation, no public save/release/telemetry/account/store
  surface change.

## Decisions and deviations

- Used the Web-local adapter from `apps/web/src/simulation-worker-session.ts`
  and WM-0160 `drainWebPlayableCommandsToTerminal` instead of the older
  browser-local advance timer pump.
- Preserved base product-gate alerts while adding authoritative Worker status
  updates so the initial player HUD keeps visible night-risk context and command
  completion still updates alerts/resources/status.
- `coordination/thread-registry.json` and
  `coordination/inbox/client-engineer/MSG-MR0WUVDI-B9E52D.json` were already
  dirty/untracked control-plane state from orchestration and were preserved.
