## Determinism, save, performance and security impact

Determinism: no runtime code changed; the contract fixes seed `1`, random stream ids, command stream, 30 TPS, integer material/progress semantics, stable end-state assertions and long-run queue invariants for later implementation.

Save: no save code changed. The contract requires carried items, build buffers, reservations, terminal build state and reason traces to be authoritative and auditable for later save/replay work.

Performance: no hot path changed. The contract prohibits global pawn scans, unbounded sorts, hot-loop allocations and sustained queue growth; it sets provisional candidate cap `8` and 100000-tick idle invariants.

Security: no dependencies, UI, Electron, preload, IPC or filesystem access changed. The contract keeps Worker/headless authority and forbids renderer-side world mutation.
