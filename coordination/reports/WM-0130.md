# WM-0130 Work report

## Outcome

WM-0130 produced the current-code blocker matrix for the post-M8 Web shell. The
visible default launch is no longer the diagnostics harness, but the runtime is
still fixture-backed, the town HUD remains mixed-language in `zh-CN`, camera
drag/reset are absent, object-level interaction is not proven in the default
fixture, and the minimum playable gameplay chain is blocked by protocol and UI
adapter gaps rather than by shell chrome alone.

## Files and public contracts changed

- `coordination/reports/WM-0130-plan.md`: audit plan and evidence sources.
- `coordination/reports/WM-0130.md`: blocker matrix, evidence, and handoff.

No product code, protocol contract, save behavior, Electron boundary, or public
release verdict changed.

## Acceptance evidence

- Default launch audit:
  The visible default path is the start surface from
  `packages/ui-react/src/shell-hud.ts`, while diagnostics visibility is gated by
  `apps/web/src/shell-locale.ts`. The runtime still seeds from
  `apps/web/src/smoke-read-model.ts`.
- Mouse interaction, layering, clickability, and empty-tile feedback:
  Pixi selection and wheel/keyboard handling are in
  `packages/renderer-pixi/src/pixi-world-renderer.ts`; shell reachability and
  selection flows are covered by `apps/web/src/web-shell.e2e.test.ts`.
- Locale default/manual switching, hardcoded strings, and diagnostics isolation:
  Locale resolution and persistence are in
  `packages/ui-react/src/localization.ts` and `apps/web/src/shell-locale.ts`.
  Mixed-language HUD leakage is evidenced by `packages/ui-react/src/shell-hud.ts`
  consuming prose-heavy `WorldReadModel` fields from
  `packages/sim-protocol/src/web-read-model.ts` and
  `apps/web/src/product-gate-fixture.ts`, plus a local runtime sample confirming
  zh-CN HUD locale with English goal/resource text.
- Responsive and screenshot evidence needs:
  Existing committed artifacts under `coordination/artifacts/WM-0118/` show
  viewport reachability, but the JSON metrics for small-window player-HUD cases
  reveal offscreen sections at baseline. Screenshot coverage remains a curated
  subset only.
- Minimum playable chain:
  `packages/sim-protocol/src/constants.ts` and
  `packages/sim-protocol/src/types.ts` show the current player-command surface
  is not gameplay-capable, and `packages/ui-react/src/shell-hud.ts` does not
  expose an actionable command bar.
