`m8.faction_endgame.owner_arcs.v1` owner slice. The new gate adds M8

## Determinism, save, performance and security impact
Determinism: M8 uninterrupted and resumed checkpoints match, and divergence
comparison reports the first divergent tick. Save: this is a focused M8
scenario envelope only, not a public Save Container or compatibility promise.
Migration: policy validation is deliberately negative and rejects unapproved
public/cross-platform claims. Performance: no per-tick runtime work is added;
the gate records bounded rows, route visits, faction-fact visits, cap hits and
stale rejects. Security: no new dependencies, no Electron/preload/platform
filesystem surface, no telemetry, account, paid-service, release or store work.

## Decisions and deviations
- Architecture decision is documented in
  `docs/02_systems/13_save_replay_migration.md` because
  `coordination/decisions/**` is outside WM-0125 allowed paths.
- `PLANS.md` was not edited because it is outside WM-0125 allowed paths; the
  execution plan was kept in-thread.

  keeps gate-local envelope types, validation and replay helpers together. The
  file is longer than the general 400-line preference; splitting should happen
  when a shared save-container abstraction is approved rather than by adding an
  unreviewed abstraction in WM-0125.
