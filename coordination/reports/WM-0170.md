# WM-0170 Blocker Report

## Outcome

Blocked before product implementation. The mandatory public-API audit found
that the existing indexed selection, exact path, ability and atomic reservation
surfaces cannot satisfy WM-0170's owner-version revalidation and zero-allocation
hot-path contract within the task's allowed files.

No autonomy implementation, generic owner, GameSession integration, protocol,
Worker, Web, save, dependency, PR-3 or release file changed.

## Blocking evidence

| Public surface | Evidence | Missing contract |
| --- | --- | --- |
| `WorkOfferIndex` | `WorkOfferInput` has no owner-version field; the class exposes no index version; `readOffer` and selection return new objects | Version-bound selected row and allocation-free caller-owned read/selection output |
| `GridPathfinder` | `findPath` allocates a new `Uint32Array` on trivial and normal successful paths | Caller-owned bounded path output/result lanes |
| `ReservationLedger` | `acquire` builds `PreparedClaim[]` and `claimIds[]` for every transaction | Caller-owned prepared transaction and claim-id scratch while preserving one atomic commit |
| `M3AbilityCacheStore` | `queryAbility` returns a newly materialized result object | Caller-owned ability result lane for the hot arbitration path |

The ReservationLedger's existing validation/commit behavior is atomic, but its
public transaction surface still allocates. Supplying an unrelated scalar
"owner version" beside a WorkOffer would not bind that version to the selected
row and would make stale rejection caller-asserted rather than owner-proven.

## Required architecture decision

Create and review a prerequisite owner-surface repair that either:

1. adds a monotonic/version-bound WorkOffer read contract and caller-owned
   `select/read/findPath/acquire/queryAbility` output/scratch lanes; or
2. explicitly revises WM-0170's zero-allocation and offer-owner-version
   acceptance contract with measured justification.

WM-0170 should remain blocked and unimplemented until that prerequisite is
verified, integrated and done. Expanding WM-0170's allowed paths ad hoc would
violate the reviewed strict-serial DAG and generic owner ownership boundary.
