# WM-0112 Work report

## Outcome

Defined the M8 player-facing product UI design system and visual identity in a
new focused design document. The spec replaces the current diagnostics-harness
default target with a player HUD hierarchy, token contract, explicit debug
overlay boundary and scenario fixtures for downstream implementation and QA.

## Files and public contracts changed

- `docs/01_design/09_m8_product_ui_design_system.md`: new M8 design-system
  contract for theme tokens, HUD hierarchy, component states, visual identity,
  explainability, debug separation and downstream scenario fixtures.
- `coordination/reports/WM-0112.md`: this report.
- `coordination/tasks/WM-0112.json`: task state/history updated by
  `taskctl claim` and `taskctl complete`.

## Acceptance evidence

- Theme tokens: `docs/01_design/09_m8_product_ui_design_system.md` defines
  color, typography, spacing, panels, buttons, alerts, resource cards, resident
  cards, Chronicle cards and debug overlays. Hex/pixel values are explicitly
  marked provisional.
- Default player HUD information hierarchy: the spec defines priority order,
  desktop and compact layout contracts, current phase meanings, next-goal data
  fields, resources, resident state, tasks/events, obligations and night-risk
  tiers.
- Wuming Town visual identity: the spec maps lantern boundaries, Chronicle,
  ordinances, residents, explanation and folk-horror settlement cues into UI
  expression without requiring final art assets.
- Debug/diagnostics boundary: the spec requires diagnostics to be explicit
  dev/debug overlays and blocks default player display of the M6 harness,
  hashes, protocol dumps, fixture names and release verdicts.
- No runtime implementation or final release claims: only allowed
  documentation/coordination paths were changed; the spec preserves Web
  `demo-only`, unsigned Windows controlled-test scope, and no public release,
  store, signing, telemetry, account, paid-service or save-compatibility claims.
