# WM-0151 Work report

## Outcome

Implemented player-readable pawn motion and job feedback in the Web/Pixi shell
without giving Web any authoritative world ownership. The renderer now shows
intent/path lines, target markers, per-job shape/state cues, and progress at
the target; the inspector explains current job, step, target, progress and
structured reasons from reviewed read-model data.

The required architecture correction was applied before finishing: the illegal
`apps/web -> packages/sim-core/src/index` imports were removed. Web now uses a
reviewed projection harness built only from legal `@wuming-town/sim-protocol`
types and local `WorldReadModel` projection data. No `package.json` boundaries
were loosened, and no UI-owned authoritative mutation was added.

## Files and surfaces changed

- `apps/web/src/reviewed-playable-session.ts`: added the legal reviewed
  projection harness for command -> marker -> pawn -> progress -> result
  playback, plus debug payload projection.

  local runtime stub to the reviewed projection harness.

## Decisions and deviations

- WM-0151 originally moved toward a local authoritative Web runtime, but that
  violated package-boundary rules. The final implementation explicitly backs
  away from that: Web stays a read-model/projection consumer and does not claim
  Simulation Worker ownership.
- Because `apps/web` has no legal public dependency path to the WM-0150 runtime
  slice, the Web harness demonstrates the reviewed player-readable feedback
  using structured projection snapshots rather than a live local `sim-core`
  runtime.
- Reviewer repair: the Web copy now consistently describes projection playback /
  reviewed harness behavior instead of implying live Worker integration or a
  reviewed authoritative slice.
- Reviewer repair: non-WM-0150 reason codes were removed; the Web playback and
  inspector now reuse reviewed codes such as `no_path` and `invalid_target`
  rather than inventing new command semantics.
- The long M8 responsive viewport matrix E2E now uses a `720000ms` timeout.
  This does not relax assertions; it only reflects the real runtime needed for
  the full viewport matrix under the required wrapper command.

## Remaining risks / follow-up

- The WM-0151 Web harness is a reviewed projection playback, not direct live
  Worker integration. A future task can replace it once a legal public Worker /
  protocol surface is exposed to `apps/web`.
- The new structured read-model fields are additive and UI-facing only; if more
  job kinds arrive, marker/icon copy and inspector wording will need extension.
- `node tools/test-runner.mjs e2e --filter web-shell` is now green, but it is a
  long-running gate. Any future viewport/artifact expansion should budget that
  runtime explicitly.
