# WM-0114 Work report

## Outcome
WM-0114 adds a narrow zh-CN/en localization runtime for the current shell, ADR-0011 locale resolution, manual override persistence scoped to shell presentation state, and a reusable localization completeness gate. Default player UI no longer exposes developer diagnostics; diagnostics remain available only through explicit debug entry points (`wmDiagnostics=1` / `WM_DESKTOP_QUERY=wmDiagnostics=1`). Simulation authority, save authority, Electron preload scope, and forbidden package boundaries remain unchanged.

## Files and public contracts changed
- `packages/ui-react/src/localization.ts`: typed locale ids, locale preference contract, zh-CN/en catalogs, ADR-0011 locale resolution, formatting helpers, persistence helpers, completeness validation.
- `packages/ui-react/src/localization.test.ts`: unit coverage for locale resolution, malformed preference recovery, persistence fallback, and localized template formatting.
- `packages/ui-react/src/shell-settings-panel.ts`: player-facing shell language settings UI and persistence diagnostics messaging.
- `packages/ui-react/src/shell-onboarding-panel.ts`: localized onboarding shell chrome and copy-limit messaging.
- `packages/ui-react/src/shell-hud.ts`: shell chrome localization wiring and default hiding of diagnostic-only surfaces.
- `packages/ui-react/src/shell-hud.test.ts`: verifies localized shell chrome and that diagnostics stay out of default HUD markup.
- `packages/ui-react/src/shell-store.ts`: shell locale state/actions and diagnostics visibility state.
- `packages/ui-react/src/index.ts`: public exports for localization runtime and settings panel.
- `apps/web/src/shell-locale.ts`: browser locale detection, localStorage persistence controller, diagnostics query parsing.
- `apps/web/src/shell-locale.test.ts`: focused tests for query-gated diagnostics and browser locale controller behavior.
- `apps/web/src/shell-bootstrap.ts`: wires locale controller into the web shell and keeps diagnostics opt-in.
- `apps/web/src/web-shell.e2e.test.ts`: preserved existing coverage, narrowed diagnostics expectations, added locale default/persistence coverage.
- `apps/desktop-electron/src/main.ts`: minimal main-process support for deterministic locale/persistence e2e setup (`WM_DESKTOP_USER_DATA_DIR`, `WM_DESKTOP_QUERY`) without broad API exposure.
- `apps/desktop-electron/src/desktop-shell.e2e.test.ts`: desktop locale default/persistence coverage and diagnostics opt-in expectations.
- `tools/validate-localization.mjs`: reusable completeness gate for player-visible localization catalogs.
- `tools/test-runner.mjs`: uses `corepack pnpm` for repo-compatible nested vitest execution.
- `package.json`: adds localization validation to repo scripts and normalizes nested pnpm invocations to `corepack pnpm`.
- `apps/desktop-electron/package.json`: normalizes nested pnpm invocations to `corepack pnpm`.
- `coordination/reports/WM-0114-plan.md`: execution plan.
- `coordination/tasks/WM-0114.json`: workflow claim state updated by taskctl.

## Acceptance evidence
- `Adds a localization layer for zh-CN and en with typed keys or equivalent validation`
  - `packages/ui-react/src/localization.ts` defines `LocaleId`, `MessageKey`, catalogs, format helpers, and catalog validation.
  - `packages/ui-react/src/localization.test.ts` and `node tools/validate-localization.mjs` both pass.
- `Defaults Chinese browser/system language to zh-CN and non-Chinese to en`
  - `resolveSystemLocale()` implements the ADR-0011 rule.
  - `packages/ui-react/src/localization.test.ts`, `apps/web/src/shell-locale.test.ts`, `apps/web/src/web-shell.e2e.test.ts`, and `apps/desktop-electron/src/desktop-shell.e2e.test.ts` cover `zh-* -> zh-CN` and non-Chinese -> `en`.
- `Adds manual language override persistence without broad Electron authority`
  - Web persistence uses renderer storage only via `apps/web/src/shell-locale.ts`.
  - Desktop coverage uses shell-local persistence with `WM_DESKTOP_USER_DATA_DIR`; no preload/API broadening was added.
  - Web and desktop e2e both prove manual override survives reload/restart.
- `Adds missing-key and translation completeness validation that fails tests`
  - `tools/validate-localization.mjs` fails on missing player-visible zh-CN/en entries and is wired into `package.json` / `pnpm quality`.
  - `packages/ui-react/src/localization.test.ts` asserts `validateLocalizationCatalogs()` returns no issues.
- `Keeps developer diagnostics isolated from default player UI`
  - `packages/ui-react/src/shell-hud.ts` only renders diagnostics when `diagnosticsVisible` is true.
  - Web/desktop e2e keep diagnostics coverage through explicit debug query flags and verify default player UI no longer shows those panels.
- `zh-CN catalog is readable UTF-8 and valid`
  - Verified `packages/ui-react/src/localization.ts` decodes correctly with explicit UTF-8 and begins with bytes `65-78-70`, confirming no UTF-8 BOM.
  - `node tools/validate-localization.mjs`, `corepack pnpm typecheck`, and `corepack pnpm quality` all pass after the verification.

## Determinism, save, performance and security impact
- Determinism: none. No simulation tick logic, authority flow, or deterministic core packages changed.
- Save: none. Locale preference persists only as shell presentation state; no save schema or public save compatibility contract changed.
- Performance: low risk. Localization lookups are static map reads and string template replacement in shell UI paths only.
- Security: low risk and reviewed. Electron remains sandboxed; no broad preload bridge, filesystem exposure, telemetry, account, store, signing, or paid-service work was added.

## Decisions and deviations
- Followed `coordination/decisions/ADR-0011.md`: `zh-*` system/browser languages resolve to `zh-CN`; non-Chinese or malformed candidates resolve to `en`; manual override persists locally as shell presentation state only.
- Diagnostics fallback is restricted to explicit debug surfaces only. Default player HUD no longer carries release-gate/storage diagnostic panels.
- `apps/web/src/web-shell.e2e.test.ts` was preserved and patched narrowly. Existing e2e coverage was not removed.
- `rapid-implementer` was not used. This task exceeded Spark lane limits because it touched web, desktop, shared UI runtime, tooling, tests, and docs.

## Remaining risks / follow-up tasks
- Current world/read-model content strings remain mostly English; this task intentionally localizes shell chrome only. Full content translation remains separate scope.
- Locale validation currently treats non-`dev` namespaces as player-visible and complete for zh-CN/en. If new namespaces are introduced later, the validator rules should be reviewed together with the catalog contract.
