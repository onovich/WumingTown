# WM-0010 Work report

## Outcome

Established reproducible local and CI quality gates for WM-0010. The repository now records exact tool/environment versions per CI job, compares benchmark medians against an explicit checked-in baseline, emits deterministic replay artifacts with seed and first divergent tick context, and enforces a no-silent-retry policy through both configuration and a dedicated guard script.

## Files and public contracts changed

- `.github/workflows/ci.yml`: split CI into local-parity, benchmark, and dependency-audit jobs on pull requests and task/main pushes, with artifact upload for each job.
- `package.json`: added `ci:local` and `audit:deps`, broadened formatting coverage, and kept benchmark/replay entry points reproducible from the root.
- `vitest.config.mjs`: made `retry: 0` explicit.
- `packages/benchmarks/baseline.json`: checked-in explicit benchmark baseline used by `pnpm bench`.
- `packages/benchmarks/src/benchmarks.ts`: moved benchmark execution into reusable suite/sample helpers.
- `packages/benchmarks/src/baseline.ts`: added baseline comparison and regression classification logic.
- `packages/benchmarks/src/cli-lib.ts`: added benchmark CLI parsing, artifact writing, baseline loading, and summary output.
- `packages/benchmarks/src/index.ts`: exported the new benchmark/baseline surface.
- `packages/benchmarks/src/baseline.test.ts`: covered benchmark comparison behavior.
- `tools/ci-local.mjs`: created the local CI parity runner that mirrors workflow commands.
- `tools/write-environment-report.mjs`: records exact Node/pnpm/npm/OS/CPU/git metadata to JSON artifacts.
- `tools/audit-deps.mjs`: wraps `pnpm audit --json` and preserves audit artifacts.
- `tools/assert-no-test-retries.mjs`: fails if non-zero retry configuration is introduced.
- `tools/sim-replay-test.mjs`: upgraded replay diagnostics to long-run checkpoints with artifact emission and structured divergence output.
- `docs/05_tech/03_performance_budget.md`: documented the explicit benchmark baseline path and regression thresholds.
- `docs/05_tech/05_testing_observability_ci.md`: documented local CI parity commands, environment artifacts, determinism failure outputs, and the retry policy.

## Acceptance evidence

- CI runs on pull requests and records exact environment/tool versions:
  `.github/workflows/ci.yml` now triggers on `pull_request`; each job runs `node tools/write-environment-report.mjs` and uploads artifacts. Local evidence path: `coordination/artifacts/WM-0010/environment/tool-versions.json`.
- Determinism and long-run failures emit seed, first divergent tick and artifact paths:
  `pnpm sim:replay-test` now writes replay artifacts and prints the seed plus first divergent tick for the perturbed probe. Local artifact paths: `coordination/artifacts/WM-0010/determinism/expected-checkpoints.json`, `coordination/artifacts/WM-0010/determinism/actual-checkpoints.json`, `coordination/artifacts/WM-0010/determinism/perturbed-checkpoints.json`.
- Benchmark changes are compared to an explicit baseline:
  `pnpm bench` loads `packages/benchmarks/baseline.json`, compares sample medians, and writes `coordination/artifacts/WM-0010/benchmarks/benchmark-results.json`.
- No test silently retries nondeterministic product behavior:
  `vitest.config.mjs` sets `retry: 0`, and `node tools/assert-no-test-retries.mjs` passes while scanning repo test/CI sources for non-zero retry configuration.

## Determinism, save, performance and security impact

Determinism: improved through structured long-run replay artifacts, stable seed reporting, and explicit first-divergence output.

Save: none. No save/load format or persistence contract changed.

Performance: benchmark enforcement added with explicit baseline comparison and documented warning/fail thresholds; no sim-core hot-path behavior changed.

Security: improved dependency-audit visibility and CI environment traceability; no new runtime dependency introduced.

## Decisions and deviations

- Kept benchmark enforcement in the existing root `pnpm bench` entry point instead of creating a separate task-only command, to preserve local/CI parity.
- Chose stable generated artifact paths under `coordination/artifacts/WM-0010/` so reviewers can rerun commands without changing script arguments.
- Did not add framework-level retries to reduce flake; instead added a guard that fails if non-zero retry configuration is introduced later.

## Remaining risks / follow-up tasks

- GitHub Actions execution was validated by workflow inspection and exact local command parity, but not exercised against the remote GitHub service from this workspace.
- `tools/write-environment-report.mjs` and `tools/audit-deps.mjs` still emit Node's Windows `DEP0190` warning because fixed shell-backed invocations are currently the most reliable way to query CLI versions in this environment. This is noise, not a functional failure.
- Benchmark thresholds are based on the current checked-in baseline for this task branch. If hosted CI hardware changes materially, the baseline may need a deliberate refresh rather than threshold drift.
