# WM-0086 Work Report

## Outcome

Added a repeatable Web product-gate harness on top of the existing Web shell.
The shell now boots a deterministic 192x192 M5/M4-derived fixture instead of
the old tiny smoke map, exposes Chrome/Edge and cross-origin-isolation
assumptions in the UI/debug surface, and writes a build-time release-gate
report during `pnpm build:web` without changing simulation authority.

## Files And Public Contracts Changed

- `apps/web/src/product-gate-fixture.ts`: deterministic 192x192 Web fixture
  with 40 visible actors and M5/M4-rooted inspector/town facts.
- `apps/web/src/product-gate-harness.data.json`: single-source harness
  assumptions for browser targets, bundle budget, asset policy and SAB/COI
  fallback.
- `apps/web/src/product-gate-harness.ts`: typed harness descriptor and
  read-only HUD/debug state mapping.
- `apps/web/src/shell-bootstrap.ts`: threads runtime browser/isolation data
  into the shell state and debug payload.
- `apps/web/src/web-shell.e2e.test.ts`: validates the richer product-gate
  harness instead of the old tiny shell-only surface.
- `apps/web/package.json`: Web build now emits the release-gate report after
  Vite build.
- `tools/web-release-gate-report.mjs`: scans the built renderer output and
  writes `wm-release-gate-report.json` with artifact sizes and recorded
  assumptions.
- `packages/ui-react/src/shell-store.ts`, `shell-hud.ts`, `index.ts`,
  `shell-hud.test.ts`: HUD and state support for the read-only release-gate
  metadata.
- `docs/07_roadmap/04_web_release_gate.md`: WM-0086 harness baseline note.
- `coordination/reports/WM-0086-plan.md`: required implementation plan.
- `coordination/tasks/WM-0086.json`: taskctl claim metadata only.

No `packages/sim-core/**` files changed. No Worker protocol, save format,
runtime dependency, server requirement or UI authority change was added.

## Determinism, Save, Performance And Security Impact

- Determinism: no authoritative simulation code changed. WM-0086 only consumes
  reviewed M5/M4 evidence and deterministic fixture data.
- Save: no save/load, OPFS, import/export or compatibility promise changed.
- Performance: WM-0086 does not claim Chrome/Edge pass/fail performance. It
  creates the repeatable fixture/report baseline for WM-0087. The only current
  build warning is Vite's chunk-size warning for the main JS chunk.
- Security: no new dependency or server requirement was introduced. The harness
  records same-origin / CORP / CORS expectations before any future SAB path.

## Decisions And Deviations

- The legacy `WEB_SHELL_SMOKE_READ_MODEL` export is kept as a compatibility
  alias, but it now points at the richer WM-0086 product-gate fixture.
- The release-gate report is written into the built renderer output directory so
  `pnpm build:web` remains the single repeatable harness command and Electron's
  later packaging flow can reuse the same built artifact root.
- The Web shell e2e screenshot path now resolves to a temp or
  `WM_ARTIFACT_DIR` artifact root so the required Playwright gate leaves the
  tracked repository artifacts untouched.

## Remaining Risks / Follow-Up Tasks

- WM-0087 must decide whether the current Web chunk split is acceptable or
  whether the large main bundle needs product-gate action.
- WM-0088 and WM-0089 still own Web storage and SharedArrayBuffer-unavailable
  behavior beyond the assumptions recorded here.
- The current Pixi renderer remains a fixture consumer, not a proven
  high-scale release renderer. WM-0086 intentionally does not over-claim
  performance readiness.
