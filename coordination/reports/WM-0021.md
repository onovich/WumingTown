# WM-0021 Work report

## Outcome

Implemented incremental region/room rebuild support in `sim-core`. `MapGrid`
now carries deterministic cardinal wall and closed-door topology lanes, and
`RegionRoomRebuilder` owns budgeted dirty-cell rebuilds, room/region id writes,
monotonic navigation/region/room/graph versions, structured reason codes,
metrics, snapshots and a versioned region-graph basis view for WM-0022 pathing.

## Determinism, save, performance and security impact

- Determinism: topology and rebuild state use integer lanes, typed arrays,
  stable FIFO dirty queues and deterministic cardinal neighbor order. No ambient
  randomness or real-time authority was added to `sim-core`.
- Save/replay: no save container changes. New snapshot/basis fields are public
  inputs for later save/replay work but are not persisted yet.
- Performance: normal invalidation is local; rebuilds consume caller budget and
  report backlog. Benchmark confirms no remaining dirty cells or active backlog.
- Security: no DOM, React, Pixi, Electron, Node filesystem or new runtime
  dependency was added to `sim-core`.

## Decisions and deviations

- ADR-0003 followed: derived region/room ids stay in `sim-core`, no singleton
  or UI authority.
- ADR-0006 followed: monotonic versions are exposed for stale async rejection,
  and rebuild work is deterministic and budgeted.
- Region graph is component-based in WM-0021. `edgeCount` is `0` because each
  rebuilt passable component is represented as one region; WM-0022 can consume
  region id equality plus graph/navigation versions before adding path search.
- Spark was not used; this task changes deterministic simulation architecture
  surfaces and benchmark plumbing.

## Remaining risks / follow-up tasks

- Region ids are monotonic and not compacted. Save/load migration can decide
  whether to compact during explicit load rebuild.
- Entities already indexed by old region ids are not migrated here; WM-0020
  spatial indexes remain derived from current placement and future movement.
  WM-0022/WM-0024 should treat navigation basis versions as stale-rejection
  inputs before consuming old region memberships.
- The region graph basis is intentionally minimal; WM-0022 owns path requests,
  path search and any coarse graph edge expansion.
