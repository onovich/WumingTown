# WM-0057 Work report

## Outcome

Implemented a focused M3 save/load/resume replay harness for
`m3.ordinary_life.injured_caregiver.v1`. The harness saves at tick `12000`,
loads at tick `12001`, resumes through tick `36000`, validates focused envelope
shape before load success, rebuilds named derived surfaces by hash, and emits
M3 replay diagnostics artifacts. No public save container, platform save UI,
schema migration, codec dependency, Worker protocol change, or M4 scope was
added.

Repair after reviewer `changes_requested`: `resumeM3OrdinaryLifeFromSave` now
hydrates a focused resume state from the validated envelope, reuses the loaded
save checkpoint/projection at tick `12000`, verifies post-load command/content
prefix continuity before producing resumed checkpoints, and marks resumed runs
with `source: loaded-save` plus `loadedStateHash`. Diagnostics now compare
resumed output against the full uninterrupted replay filtered to matching ticks,
including final read-model hash.

Second repair after re-review: the hydrated post-save checkpoint path no longer
calls `runM3OrdinaryLifeScenario`. Uninterrupted replay and save creation still
use the scenario runner, but `advanceHydratedM3ResumeState` now gates resumed
checkpoints on the loaded focused envelope fingerprint, command tail, queue
metrics, performance counters, invariants, and rebuilt save checkpoint state.
The unit suite adds a regression where a shape-valid loaded save has a tampered
queue metric: load reaches the focused resume gate, but resume rejects instead
of silently replaying from seed.

## Determinism, save, performance and security impact

Determinism: adds focused replay comparison and command ids for M3 ordinary
life, and repaired diagnostics compare resumed output to the full uninterrupted
checkpoint basis. Save: focused harness only, no public compatibility or
migration promise; resumed runs now record `loaded-save` provenance from the
validated focused envelope.
Performance: diagnostics report save bytes `4941`, rebuild time `1` tick,
checkpoint hashes, and rebuilt surface hashes; no pawn think loop full-world
scan was added. Security: no new dependency, filesystem use, UI authority,
Worker protocol, or platform save API change in sim-core.
