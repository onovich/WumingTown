## Determinism, save, performance and security impact

Determinism: ADR-0003 preserves Worker/headless authority, generation-checked entity identity, stable owner-store iteration, explicit job state, and deterministic cleanup hooks. No code changed.

Save: no save format or persistence code changed. ADR-0003 defers container byte layout to ADR-0005 and requires versioned owner stores or rebuildable derived indexes.

Performance: no benchmark baseline changed. ADR-0003 proposes typed-array owner stores, dirty queues, versioned derived indexes, Top-K candidate caps, and forbids hot-loop allocations, unbounded scans and unversioned caches.

Security: no dependency, IPC, filesystem, Electron, network, or UI mutation surface changed.
