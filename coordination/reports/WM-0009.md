# WM-0009 Work report

## Outcome
Implemented a small content validation and Def compilation spike with structured source-located diagnostics, deterministic ordering, declarative smoke content, explicit patch-conflict reporting, post-patch semantic revalidation, and JSON5 fixture handling.

## Files and public contracts changed

- `packages/content-schema/src/content-fixtures.ts`: content fixture loading, schema checks, source-location extraction, duplicate ID detection, missing reference checks, missing localization-key checks, patch-conflict detection, patch field diagnostics, and JSON5 parsing support for fixture sources.
- `packages/content-schema/src/index.ts`: public schema package exports.
- `packages/content-schema/src/content-fixtures.test.ts`: validation failure coverage for invalid ID, duplicate ID, missing reference, and missing localization key.
- `packages/content-compiler/src/compiler.ts`: validated fixture compilation into a frozen runtime catalog with stable ordering and post-patch reference/localization validation.
- `packages/content-compiler/src/index.ts`: public compiler exports.
- `packages/content-compiler/src/compiler.test.ts`: stable-order, patch-conflict, post-patch validation, invalid patch field, and JSON5 compile coverage plus core-smoke compile coverage.
- `tools/content-cli/src/index.ts`: content CLI option parsing and compilation runner.
- `tools/content-cli/src/cli.ts`: executable CLI entrypoint.
- `tools/content-validation.mjs`: workspace validation plus content-fixture validation across `sources/content/*`.
- `tools/test-runner.mjs`: added `--filter content`.
- `package.json`: added `content:compile` and updated `content:validate` to use the TS loader.
- `sources/content/core-smoke/**`: declarative anomaly/incident smoke content, localization, and a non-conflicting patch.
- `coordination/reports/WM-0009-plan.md`: execution plan.

## Acceptance evidence

- Invalid IDs, duplicate IDs, missing references, localization keys, and invalid patch mutations fail with source locations:
  - Covered by `packages/content-schema/src/content-fixtures.test.ts`.
  - Diagnostics include file path plus line/column.
- Patch-applied missing references and localization keys are revalidated after patch application:
  - Covered by `packages/content-compiler/src/compiler.test.ts`.
  - Patch diagnostics are reported against the patch source file, not silently accepted.
- Unknown or wrong-type patch fields fail instead of being ignored:
  - Covered by `packages/content-compiler/src/compiler.test.ts`.
- JSON5 source content is parsed for the smoke fixture shape instead of being silently skipped:
  - Covered by `packages/content-compiler/src/compiler.test.ts`.
- Compiler output ordering stable:
  - Covered by `packages/content-compiler/src/compiler.test.ts`.
  - Catalog definitions are sorted by ID before emission.
- Sample anomaly and incident content compile without custom runtime code:
  - Covered by `pnpm content:compile -- --fixture core-smoke`.
  - Output is the frozen runtime catalog built from declarative JSON content.
- Patch conflicts reported rather than silently accepted:
  - Covered by `packages/content-compiler/src/compiler.test.ts`.
  - Conflict reporting surfaces `patch_conflict` diagnostics instead of overwriting.

## Determinism, save, performance and security impact

Determinism: content output is sorted by ID and locale name before catalog emission. Save/replay: no save-format changes. Performance: this spike is file-driven and non-hot; no per-tick runtime cost added. Security/runtime boundaries: no arbitrary code mod support, no network access, and no new runtime dependency.

## Decisions and deviations

- Diagnostics are structured with file/line/column and avoid silent fallback when content references, patch targets, or patch field shapes are missing.
- JSON5 support is intentionally minimal and fixture-only; it handles the current smoke content shape without adding a runtime dependency.
- Patch conflict detection is exact-value based per target/path pair.

## Remaining risks / follow-up tasks

- The compiler currently handles the smoke fixture shape and a small declarative surface. A later task should expand the content schema and add more fixture coverage if more content families are introduced.
- Localization completeness is validated against the loaded locale bundles; future language support may want locale-specific acceptance rules.
- If JSON5 syntax beyond the currently supported fixture subset is required, add an approved parser dependency or a dedicated parser task.
