# WM-0133 Work report

## Outcome
Implemented the post-M8 shell design-token layer and applied it to the player HUD,
main menu, settings panel, alerts, resource cards, resident rows, inspector
surfaces and command-bar placeholder slots. The change keeps command buttons as
explicit non-authoritative placeholders for later gameplay tasks, keeps
diagnostics separate, and restores the responsive reachability gate after the new
visual surfaces initially exposed compact/720px layout pressure.

## Files and public contracts changed
- `packages/ui-react/src/shell-design-tokens.ts`: adds implementation-facing
  tokens, semantic slot names and reusable style helpers for shell surfaces,
  buttons, alerts, resource trends, select controls, hairlines and night-risk
  cards.
- `packages/ui-react/src/shell-hud.ts`: applies tokens to HUD panels, alert
  strip, resource cards, resident rows, inspector, debug overlay and disabled
  command placeholder slots; constrains compact scrolling so controls remain
  reachable.
- `packages/ui-react/src/shell-main-menu-surface.ts`: aligns start/settings
  surface chrome and buttons with the tokenized shell visual language.
- `packages/ui-react/src/shell-settings-panel.ts`: tokenizes settings surfaces
  and compacts settings controls so 720px and large UI-scale viewports remain
  reachable.
- `packages/ui-react/src/localization.ts`: adds zh-CN/en command placeholder
  labels and non-authoritative explanations.
- `packages/ui-react/src/shell-hud.test.ts`: adds markup assertions for command
  placeholder slots and zh-CN coverage.
- `apps/web/src/web-shell.e2e.test.ts`: adds alert/command slot coverage and
  preserves the no-unrelated-overlay reachability gate for HUD panels.
- `coordination/reports/WM-0133-plan.md`: records the initial execution plan.
- `coordination/tasks/WM-0133.json`: records the claim/complete workflow state.

## Acceptance evidence
- Defines implementation-facing tokens for color, typography, spacing, panel,
  button, alert, resource card, resident card, inspector and command-bar surfaces:
  `packages/ui-react/src/shell-design-tokens.ts`, plus markup/E2E assertions for
  `data-ui-slot` values.
- Uses CSS/React placeholders compatible with the asset replacement contract and
  avoids hardcoded final art images: implementation is token/CSS only; no new art
  assets, dependencies or external licenses were added.
- Improves visual identity toward wood, paper, ink, lamp light and civic record
  language: tokenized surface palettes and component helpers now drive HUD,

  release/marketing layout: `node tools/test-runner.mjs e2e --filter web-shell`

  checks.
- Adds focused tests/E2E assertions where style states affect
