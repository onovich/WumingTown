## Determinism, save, performance and security impact

- Determinism: scenario derives authoritative seed `155`, uses named random
  streams only through the season event pool and records stable content,
  command, final world and read-model hashes.
- Save: no save format, focused save/replay implementation or public save
  compatibility work was added; WM-0081 remains the stop sign.
- Performance: scenario records bounded candidate visits, cap hits, cooldown
  writes, precondition failure counts and queue-style metrics without changing
  benchmark baselines; WM-0083 remains responsible for baselines.
- Security: no new dependencies, executable content, filesystem reads, network
  effects, UI authority or public Worker protocol changes were added.

## Decisions and deviations

- ADR-0010 is followed: content definitions are data inputs and mutable state
  remains in owner stores.
- The scenario imports the WM-0079 accepted alpha catalog from
  `@wuming-town/content-schema`, which is already an allowed sim-core input.
- Review repair: the scenario now derives the M5 content basis from
  `validateM5ContentPack()` and the same stable manifest, compiled-definition
  and counter shape used by the reviewed content compiler. This records
  canonical content manifest hash `0xe55d3015` instead of a private raw-pack
  file hash, so downstream save/replay and Worker tasks share the validated
  compiled-content identity. The read-model hash also includes this content
  hash so projection evidence is tied to the same canonical content basis.
- `packages/sim-core/src/m5-alpha-content-scenario.ts` is intentionally larger
  than 400 lines because WM-0080 is an integrated evidence scenario rather than
  a reusable owner store. The mutable owner surfaces remain split across the
  reviewed M5 stores.
- No Spark lane was used; task packet marks WM-0080 ineligible because this is
  cross-surface deterministic simulation composition.
