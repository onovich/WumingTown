# WM-0148 Work report

## Outcome

The Web world view now projects a readable dusk lamp/build slice instead of a
flat terrain wallpaper. The renderer consumes additive read-model projection
data for semantic map areas, selectable markers, pawn state, intent paths,
target markers and progress cues, then draws them in Pixi without mutating
authoritative world state or changing Simulation Worker/runtime ownership.

## Files and public contracts changed

- `packages/sim-protocol/src/web-read-model.ts`: added optional read-model
  projection types for semantic areas, focus markers, and visible pawn
  activity state.
- `packages/sim-protocol/src/index.ts`: exported the new read-model types.
- `apps/web/src/product-gate-fixture.ts`: seeded the reviewed Web fixture with
  structure/lamp/dark-gap/blocked overlays, selectable targets, and idle /
  moving / working / blocked / completed pawn evidence.
- `packages/renderer-pixi/src/render-geometry.ts`: added pure helpers for
  semantic-area, marker, path and activity projection.
- `packages/renderer-pixi/src/render-geometry.test.ts`: covered semantic area
  bounds, marker placement, path projection and activity target geometry.
- `packages/renderer-pixi/src/pixi-world-renderer.ts`: rendered semantic map
  overlays, intent paths, target markers, progress bars, state badges, and more
  distinct hover/selection affordances from read-model data only.
- `packages/renderer-pixi/src/index.ts`: exported the new geometry helpers.
- `coordination/reports/WM-0148-plan.md`: execution plan.
- `coordination/reports/WM-0148.md`: this report.

No authoritative simulation state, command protocol semantics, Electron
security boundary, release/store/signing/telemetry/account/public-save surface,
or `sim-core` runtime implementation changed.

## Acceptance evidence

- Map reads as a game surface, not abstract wallpaper:
  `apps/web/src/product-gate-fixture.ts` now supplies semantic structure,
  lamp-coverage, dark-gap and blocked-area slices; `packages/renderer-pixi/src/pixi-world-renderer.ts`
  renders them with distinct fills, hatching, crosses, door emphasis and
  selectable corner markers on top of more legible terrain.
- Pawn visuals expose idle, moving, working, blocked and completed states:
  `apps/web/src/product-gate-fixture.ts` projects those states through additive
  entity activity data, and `packages/renderer-pixi/src/pixi-world-renderer.ts`
  draws per-pawn state badges, intent paths, target markers and progress bars.
- Selected target, hover target, intent/path preview, job marker and progress
  affordances are visually distinct and not color-only:
  `packages/renderer-pixi/src/pixi-world-renderer.ts` uses corner brackets,
  filled selection tiles, path lines with nodes, target-specific marker shapes,
  progress bars, and state-badge glyphs instead of color alone.
- Renderer remains a projection of read-model data and does not mutate
  authoritative world state:
  all new inputs are additive read-model fields in
  `packages/sim-protocol/src/web-read-model.ts`; renderer changes only read and
  draw those projections.
- Render and geometry tests cover entity/pawn state projection, marker
  placement and viewport behavior:
  `packages/renderer-pixi/src/render-geometry.test.ts` now exercises semantic
  area bounds, focus marker placement, path projection and activity target
  geometry alongside existing viewport tests.

## Determinism, save, performance and security impact

Determinism: none. The task only changes renderer-side read-model projections
and fixture data; Simulation Worker, tick rate, random streams and job logic are
unchanged.

Save: none. No save schema, migration, persistence envelope or compatibility
contract changed.

Performance: bounded renderer startup work increased because the Pixi layer now
draws semantic overlays and marker glyphs, but it still uses chunk graphics plus
per-entity/per-marker overlays only, not per-cell display objects or per-sprite
tickers.

Security: none. No Electron preload, IPC, storage permission, network,
telemetry, account or signing surface changed.

## Decisions and deviations

- The new protocol surfaces are optional additive projection fields so the task
  does not force unrelated package fixtures or imply authoritative command/job
  semantics before WM-0150 / WM-0151 land.
- `coordination/reports/WM-0135-responsive-layout.md` was temporarily modified
  by the long responsive E2E artifact writer, then restored to `HEAD` before
  completion to keep the WM-0148 change set inside task-packet allowed paths.
- The task keeps the existing shell-local playable command placeholder honest:
  map/read-model state now makes later authoritative work visible, but no new
  reviewed command protocol semantics were invented here.

## Remaining risks / follow-up tasks

- The longest Web E2E scenario is the responsive viewport artifact capture,
