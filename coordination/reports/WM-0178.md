# WM-0178 Work report

## Outcome

Blocked after the approved harness-only repair and the single permitted
repaired-lineage verification. Commit `7434d6f` makes every filtered or full
benchmark suite run in its own fail-closed Node child and records child PID,
command, exit code, raw samples, actual-file SHA-256, sidecar SHA-256, canonical
payload SHA-256 and a stable anonymized host key. Workloads, fixtures,
invariants, baseline medians, 10/20 percent thresholds, 9 samples and 2 warmups
are unchanged.

The repair is mechanically effective, but the canonical full gate still exits
one: `entity-store`, `m4-core-vertical-slice-long-run` and `map-dirty` fail.
The approved host stayed at roughly 75-94 percent aggregate CPU during the
measurements. Therefore this task cannot honestly claim a zero-exit gate or
attribute the residual failures to exact product implementations. No benchmark
was retried to obtain a pass, no waiver was used and no product file changed.

  process; validate child schema, finite samples, sample/warmup counts,

  host-key and hash evidence.

  order used by the isolated parent runner.

  distinct PID, raw/hash metadata and malformed-child fail-closed behavior.

  their generated SHA-256 sidecars and host-load evidence.

The JSON artifact keeps `schemaVersion: 1` and adds fields. This is treated as
an additive compatible extension because existing comparison payloads and
result fields are unchanged; consumers that reject unknown fields require
independent review before integration.
