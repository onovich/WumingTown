## Determinism, save, performance and security impact

Determinism: path selection uses stable queue order and deterministic Top-K tie
breaks by score, candidate id and target cell. The pathfinder reads integer map
state, uses typed scratch buffers and does not call `Math.random`, real time or
platform scheduling.

Save: no new save container section is introduced. The contract deliberately
keeps actor state at request/result metadata boundaries; later Job Driver state
should persist request sequence, target and version basis rather than long-lived
complete paths.

Performance: local A star uses reusable typed lanes and caller-supplied node
budgets. Work selection exact pathing is bounded by `maxExactPaths` over
already indexed candidates, avoiding pawn-by-world scans. `pathing-100` records
processed requests, accepted results, stale rejects, node expansions, backlog
peak/final and checksum.

Security: no new dependency, platform API, filesystem access or UI authority is
introduced. Benchmark filesystem writes remain inside the existing benchmark
CLI artifact path.

## Decisions and deviations

- Region coarse pathing is represented by version basis plus local A star for
  this task. A richer region-graph search can extend the same stale-result
  contract once graph-edge pathing exists.
- `PathRequestBatcher` owns queue metrics and stale commit accounting; callers
  remain responsible for issuing fresh requests after incompatible versions.
- Spark was not used because WM-0022 owns core deterministic pathing and
  async stale-version semantics.
- Two spawned `simulation-engineer` subagents stalled before implementation.
  Beacon took over implementation in the WM-0022 worktree and will still route
  the branch to independent `reviewer` / Mirror for final verification.

## Remaining risks / follow-up tasks

- Job Driver and WorkOffer tasks must adopt the path request API without
  persisting complete paths across incompatible versions.
- Region-graph coarse routing remains future work; current implementation is
  local grid A star guarded by the full version basis.
- Richer pawn risk and forbidden-zone costs should feed explicit walk-cost or
  candidate filtering inputs rather than hidden global pathfinder state.
