# 代码与设计评审清单

## 正确性

- 是否满足任务，不多做隐性功能？
- 边界、取消、重复、存档中断、目标销毁是否处理？
- 是否破坏稳定顺序、随机流或迁移？

## 架构

- 依赖方向是否正确？
- 权威状态是否泄漏到 UI/渲染？
- 是否新增中央 switch、全局单例或隐藏回调？
- 缓存是否有版本和失效拥有者？

## 性能

- 复杂度、候选上限和分配？
- 是否能索引、批量、错峰或局部更新？
- Worker 往返是否比计算本身更贵？

## 可解释性

- 玩家能知道失败原因吗？
- 开发者能看到候选、拒绝与版本吗？
- 文本是否与结构化事实同源？

## 安全与内容

- 外部数据是否验证？
- Electron/API 是否最小权限？
- 文化内容是否有来源与风险？

## 测试

- 测试是否验证行为而非实现细节？
- 是否有回归、场景和必要性能测试？
- 测试是否可能假阳性、依赖真实时间或随机？
