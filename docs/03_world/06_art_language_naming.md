# 视觉、语言与命名规范

## 视觉语法

- 暖色灯火、小面积高饱和异常色、大面积低饱和土木与墨黑。
- 白天强调生活细节与材质；夜间通过留白、轮廓缺失和不合理重复制造异常。
- 异类避免全都做成尖牙怪物；可用错误影子、过度整齐、缺少面部细节、比例轻微失真。
- UI 参考地方志、账簿、木牌和地图，但保持现代信息层级与可读性。

## 命名

- 地名简洁、可口述：灰渡、鹤回、旧桥、归灯巷。
- 异类名称优先来自居民观察或事件，不直接暴露真规则：借影客、第三声、无门户、回灯人。
- 代码 ID 使用稳定英文语义，不用拼音作为唯一含义：`core.anomaly.borrowed_shadow`。
- 角色名需按文化区域维护名字池，避免所有人同一风格。

## 文本声音

镇志：客观、标来源、保留矛盾。居民对话：短、具体、与职业关系有关。事件文本不替玩家下结论。

避免：故作古文、堆叠生僻字、所有角色说同一种文绉绉语言。正式中文以清晰现代叙述为主，少量时代词汇提供气质。
