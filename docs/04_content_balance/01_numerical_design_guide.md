# 数值设计指南

所有数值为首轮原型基线，不是最终平衡。

## 统一尺度

- 需求、情绪、能力：0–1000 整数。
- 百分比：以 basis points 或千分比整数表达。
- 时间：Tick；内容文件可写 `gameHours`，编译期转换。
- 质量/技能：0–20 显示层级，内部可用 0–2000 经验。
- 严重度：0–1000；跨阈值触发阶段。
- 关系：多维，不压为一个总值；好感可 -1000–1000。

## 数值层级

1. **基线**：普通健康成人、普通工具、白天、安全环境。
2. **乘区**：技能、工具、能力、环境、品质。乘区数量保持少，防爆炸。
3. **硬限制**：技能门槛、身体能力、镇规、路径、材料。
4. **软成本**：时间、疲劳、风险、情绪、债务。

## 工作时间目标

- 微任务：5–20 游戏分钟；避免频繁寻路成本高于工作。
- 日常任务：0.5–3 游戏小时。
- 建筑/研究：1–10 人日，通过多人或阶段完成。
- 仪式/实验：有准备期、执行窗口和恢复期，不只是长进度条。

## 资源安全库存

UI 以“可维持天数”优先，而非只显示数量：食物、灯油、药物。早期目标：食物 6–10 天、灯油 3–5 夜仍有压力但可恢复。

## 情绪

中性目标约 600。轻度风险 <400，重度 <250，极端 <120；实际阈值受人格和状态调整。单个常见因素不应直接从稳定状态推到极端。

## 知识可信度

内部支持分 0–1000，仅用于排序。建议证据基线：传闻 80、单一证词 150、独立证词 180、物理痕迹 220、控制观察 300、重复验证额外 180；利益冲突、感知缺陷、非独立来源按比例衰减。确认规则还要求来源类型多样性，不只堆数量。

## 导演压力

压力分解到外部威胁、稀缺、伤病、社会、异常和债务六维。任何单维连续两次重大事件后增加主题疲劳，降低同类权重。

## 平衡原则

- 不依赖 1% 小数差异制造选择。
- 每个强收益有可理解成本；成本可以延迟，但必须可追踪。
- 失败原因应主要来自计划和信息，而非不可见随机。
- 随机区间控制在不颠覆玩家准备的范围；重大随机必须可被保险、冗余或撤退缓解。
