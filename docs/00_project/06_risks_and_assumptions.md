# 风险与假设

## 最高风险

### R1：规则型异类变成猜谜惩罚

缓解：每种规则必须有证据预算、反证、低风险实验和事故复盘；测试玩家能否在不读攻略时形成合理假设。

### R2：TypeScript 后期模拟性能不足

缓解顺序：修正复杂度 → 建索引 → 分频/错峰 → TypedArray/Scratch Buffer → Worker 批处理 → 单个 Rust/Wasm Kernel。禁止未经测量直接重写。

### R3：角色 AI 和 Reservation 边界错误

缓解：显式状态机、事务预订、租约、世界不变量、随机场景测试、百万 Tick Headless。

### R4：内容成本失控

缓解：异类由规则组件组合；MVP 只做 12–15 种；每种必须复用通用触发、感知、痕迹、交易和后果系统。

### R5：文化表达失真或廉价化

缓解：虚构政治与宗教体系；现实素材必须记录来源与改编；文化顾问审查；避免“东方神秘主义素材拼贴”。

### R6：多代理并行导致架构漂移

缓解：任务 DAG、模块所有权、ADR、独立评审、文件型邮箱、主集成线程和严格公共 API。

## 核心假设

- 目标玩家愿意阅读解释性 UI 和镇志条目。
- WebGL2 足以支撑主要 2D 视觉；WebGPU 非必需。
- 真实玩法角色规模在 8–24，100 角色主要用于技术余量。
- 单人游戏允许确定性优先于物理真实性。
- 大部分内容可用 Def/规则图表达，而非每项编写专用代码。
