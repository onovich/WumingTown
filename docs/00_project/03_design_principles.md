# 设计原则

## 1. 规则先于剧情

剧情节点应调用稳定系统，而不是在脚本中直接加减任意状态。即使是重大事件，也应通过生成实体、伤病、记忆、债务、镇规或空间变化影响世界。

## 2. 不完全信息，但必须可推断

未知可以制造紧张；无迹可寻的随机答案只会制造不信任。每种危险规则必须拥有：前兆、痕迹、误导来源、至少一种安全验证方式和事后解释。

## 3. 代价优于唯一正确答案

处置异类、接纳旅人、制定宵禁不应只有显然最优解。不同方案改变资源、人物关系、知识可信度、派系态度与长期债务。

## 4. 角色不是棋子

角色服从工作优先级，但会因生理极限、家庭、信念、恐惧和紧急事件改变行为。任何偏离都必须产生可见原因。

## 5. 系统交叉胜过内容数量

一种能影响灯火、物流、情绪、镇规和契约的异类，优于五种只有不同伤害数值的敌人。

## 6. 玩家管理制度，不负责每一步点击

玩家表达目标和政策；角色执行。紧急微操存在，但不是正常生产的唯一方式。

## 7. 可解释性属于核心玩法

原因链、证据来源、工作拒绝、路径失败、债务到期和事件评分都应可检查。开发调试与玩家界面共享同一结构化事实来源。

## 8. 恢复空间

灾难应迫使玩家改变计划，而非连续叠加直到不可逆失败。导演需要识别当前压力、近期损失和恢复能力。
