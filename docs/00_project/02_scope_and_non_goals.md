# 范围与非目标

## MVP 必须有

- 固定 Tick 模拟、暂停与 1×/3×/6×速度
- 角色需求、工作、路径、预订、库存、搬运与生产
- 基础伤病、疼痛、能力和情绪
- 灯火空间网络与黄昏准备流程
- 镇志证据、假设和居民自动规则
- 至少一种可调查、可误判、可谈判的异类
- 结构化事件导演与恢复窗口
- 分块版本存档、重放输入与世界 Hash
- Headless 场景、性能基准、开发覆盖层与失败原因 UI

## 本体 v1 可以有

- 12–15 种异类、多个季节与区域事件
- 3 个人类派系、1 个异类社会
- 镇规、议事与机构路线
- 家庭、师徒、恩情、血债和身份关系
- 多种终局秩序
- 数据模组、内容补丁、本地化

## 明确不在首版

- 自由文本 LLM 角色对话
- 完整世界旅行与多基地实时模拟
- 多层建筑、地下城或流体物理
- 数千单位战争
- 真实宗教教义模拟
- 修仙境界、法宝战力和宗门争霸
- 复杂遗传系统
- 联机、PvP、云服务依赖
- 任意 JavaScript 代码模组
- 原生 macOS 首发承诺

## 防止范围膨胀的判断

新功能只有在满足至少两项时才进入当前里程碑：

1. 加强三大核心支柱之一；
2. 与至少三个现有系统形成可验证交叉；
3. 能在一张截图或 20 秒视频中显著表达产品差异；
4. 提高重复游玩，而不是只贡献一次性文本；
5. 已有技术与内容预算能够覆盖测试和解释 UI。
