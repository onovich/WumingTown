# 对标机制研究结论

本项目吸收的是殖民模拟的架构原则，不复制任何具体实现。

## 关键机制结论

1. **Def 驱动**：静态定义、运行时实例与可选组件分离，内容主要通过数据扩展。
2. **固定 Tick 与分频更新**：移动/战斗高频，需求/健康/导演低频，实体按 ID 错峰。
3. **并行空间场**：地图不是 Tile 对象数组，而是地形、占用、房间、温度、光照、路径等连续数据与局部脏区。
4. **Job 与执行分离**：决策产生 Job，Job Driver 以显式状态机执行，能够中断、恢复和保存。
5. **Reservation 是事务**：多目标先检查再原子获取，失败或取消统一释放。
6. **生产命令是声明式目标**：玩家设置订单、过滤器、库存阈值，由工作系统寻找人和材料。
7. **事实—思想—情绪—行为**：客观事件不直接触发精神结果，中间层使后果可解释。
8. **统一健康条件**：伤病、疾病、植入与缺失部位通过条件系统影响派生能力。
9. **战斗价值在后果链**：伤病、物流、医疗、关系和工作能力比单纯命中公式重要。
10. **导演控制节奏，不写剧情**：读取压力与恢复能力，选择合法事件并避免连续不可恢复打击。
11. **存档与调试从第一天设计**：跨引用解析、缓存重建、版本迁移、AI 失败解释和覆盖层不可后补。

## 洁净室约束

执行代理不得从反编译仓库复制代码段、类名体系、注释、常量表或内部文本。架构文档给出的命名已是本项目独立抽象；如需进一步研究，先写“问题清单”，由研究代理归纳行为，再由另一实现代理独立设计。
