# 术语表

| 术语 | 定义 |
|---|---|
| 居民 / Pawn | 具有需求、技能、关系、记忆、工作和健康的长期角色。代码默认使用 `Pawn` 或 `Resident`，最终统一前不得混用。 |
| 异类 / Anomaly | 不完全遵守普通人类秩序、但具有稳定可调查规则的存在、现象或群体。 |
| 灯火 / Lantern Network | 划定人类可居住与可解释空间的设施、燃料、守灯职责和空间场。 |
| 镇志 / Chronicle | 保存传闻、证词、观察、假设、确认规则和制度知识的系统。 |
| 证据 / Evidence | 支撑或反驳某项假设的结构化事实。 |
| 镇规 / Ordinance | 由玩家制定、居民可理解并自动执行的公共规则。 |
| 旧债 / Obligation | 对人物、派系、异类或地点承担的有期限、有条件义务。 |
| 契约 / Pact | 明确双方、条件、收益、违约和持续时间的旧债组合。 |
| 事实 / Fact | 已在模拟中发生并可供系统消费的结构化事件。 |
| Def | 静态内容定义；运行时实例不得把内容参数硬编码在逻辑中。 |
| Job | 角色当前意图与参数。 |
| Job Driver | 将 Job 拆为显式可保存步骤的状态机。 |
| Reservation | 角色对实体、格子、数量或工作位置的临时事务性占用。 |
| Work Offer | 由工作产生方注册的可索引工作机会。 |
| Story Director | 根据殖民地压力和叙事新鲜度选择事件节奏的系统。 |
| Authoritative State | 唯一可以决定游戏结果、保存和重放的模拟状态。 |
| Render Snapshot | 从权威状态投影的只读表现数据。 |
| UI Read Model | 为界面准备的增量、可订阅、非权威数据。 |
| Dirty Queue | 记录局部失效并按预算重建缓存的队列。 |
| Vertical Slice | 同时覆盖模拟、表现、UI、内容、存档和测试的一条完整玩法链。 |
