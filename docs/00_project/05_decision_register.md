# 决策登记

## 已锁定

| 编号 | 决策 | 理由 |
|---|---|---|
| D-001 | 项目名使用《无明镇》 | 题材识别明确，承载“失名、灯火、未知秩序”主题。 |
| D-002 | 采用东方志怪边境聚落，而非修道院方向 | 差异化、系统交叉和视觉传播更强。修道院保留为替代概念档案。 |
| D-003 | TypeScript-first Web 技术栈 | AI 可读写、CLI 可验证、UI/内容工具效率高，反馈环短。 |
| D-004 | Windows + Web 优先；原生 Mac 延后 | 减少首发签名、公证和平台测试成本，同时不牺牲未来可移植性。 |
| D-005 | Electron 桌面壳 | 固定 Chromium/Node 运行时，避免系统 WebView 差异；游戏包体不是主要约束。 |
| D-006 | PixiJS 渲染 + React UI | Pixi 负责高性能 2D 世界，React 负责数据密集型界面，职责清晰。 |
| D-007 | Dedicated Worker 权威模拟 | 主线程稳定、并发边界明确、易于 Headless 与重放。 |
| D-008 | 不全面采用 Rust/Wasm | 优先开发效率；只迁移经测量确认的纯计算热点。 |
| D-009 | 三支柱：灯火、镇志、旧债 | 让题材进入模拟核心，而不是美术换皮。 |
| D-010 | 数据模组优先、代码模组延后 | 保持 Web、安全、确定性和性能边界。 |

## 需要 ADR 才能改变

- 权威模拟线程模型
- 语言或渲染引擎
- 包边界和存档主格式
- 首发平台等级
- 允许代码模组
- 改变 30 TPS
- 引入后端强依赖
- 移除重放或 Headless

## 尚待验证

- TypeScript 在真实压力场景中的快进性能
- Pixi 大量实体和 Chunk 更新策略
- SharedArrayBuffer 是否必要
- Web 内存与存档配额
- 首个存档容器压缩方案
- 镇志“未知规则”对玩家是否公平且可读
