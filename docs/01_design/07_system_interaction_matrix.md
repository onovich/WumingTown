# 系统交叉矩阵

| 系统 | 影响 | 被影响 |
|---|---|---|
| 灯火 | 夜间可达性、情绪、物流、异类触发、镇域 | 燃料、天气、守灯人、债务、建筑布局 |
| 镇志 | 居民自动行为、调查效率、镇规、处置方案 | 证词、痕迹、实验、政治删改、档案损坏 |
| 旧债 | 事件池、派系、资源、居民关系、终局 | 交易、救援、违约、继承、隐瞒 |
| 工作系统 | 生产、医疗、调查、守夜、建造 | 技能、需求、区域、路径、镇规、Reservation |
| 健康 | 工作能力、情绪、医疗、家庭故事 | 战斗、疾病、异类、环境、药物 |
| 社交 | 合作、违规、证词可信度、派系 | 共同经历、亲属、债务、镇规、死亡 |
| Story Director | 事件节奏、恢复窗口、新鲜度 | 当前压力、近期损失、季节、债务、未解案卷 |
| 镇规 | 自动行为、自由、效率、合法性 | 镇志、政治路线、居民认同、危机经验 |
| 物流 | 食物、灯油、医疗、仪式材料 | 路径、预订、仓储、危险区域、夜间规则 |

任何重大系统设计必须在规格中填写：输入、输出、失效信号、可解释原因、存档数据、性能复杂度和与至少三套系统的交叉。
