# 策划设计总纲

## 游戏是什么

《无明镇》是一款可暂停、可倍速、以间接控制为主的单人殖民地模拟。玩家管理一个会形成自身记忆与制度的共同体，而不是控制一支绝对服从的军队。

镇民白天从事农业、手工业、医疗、搬运、贸易、抄录和建设；黄昏收拢人员、点灯、封锁危险区域并安排守夜；夜间则在有限信息下应对异类规则、来客、失踪、梦游、讨债和空间异常；黎明后复盘证据、修订镇志并承担社会后果。

## 核心问题

> 一个普通人的社区，如何在无法完全理解的世界中维持秩序，同时又不因为恐惧变得残酷？

## 设计支柱

### 灯火是空间秩序

灯火同时决定照明、安全感、夜间物流、异类可达性、镇域归属和某些仪式条件。灯不是炮塔，也不是单纯供电网络。

### 镇志是可犯错的公共知识

知识由来源、证据、假设、反证和可信度组成。居民的自动行为依赖镇志；错误知识会被社会执行，因此调查与记录本身具有风险。

### 旧债是跨时间的关系

交易不只以货币结算。恩情、名字、记忆、祭礼、保护承诺和土地使用都能成为义务。短期解法可能在多年后改变社区。

### 人物与制度共同演化

个人会死亡或离开，但由玩家建立的镇规、档案、债务和文化会留给下一代。长期主角是无明镇本身。

## 核心体验节奏

```text
安定经营 → 发现异常前兆 → 信息不足下准备 → 夜间处置
→ 黎明复盘 → 社会争论 → 修订知识与制度 → 新的日常
```

危机不能天天高强度。需要“生活—疑点—准备—事件—恢复”的张弛。

## 主要玩家动词

- 规划：建造、区域、道路、灯网、仓储、工作台
- 组织：工作优先级、班次、守夜、紧急任务、封锁
- 调查：收集证词、观察痕迹、比较档案、设计实验
- 制度化：编写镇志、发布镇规、登记身份、规定葬仪
- 交涉：贸易、求援、缔约、偿债、隐瞒或公开
- 处置：隔离、疏散、引导、驱逐、封印、战斗、接纳
- 复盘：查看因果链、追责、安抚、赔偿、重建

## 失败与胜利

游戏允许局部失败、人物死亡、制度变坏和债务累积。终局不是单一 Boss，而是玩家在旧契约失效时选择一种新的区域秩序。可能结局：人间之镇、共居之镇、守秘之镇、无灯之镇、组织迁离。
