# 内容与模组总体设计

## 内容单位

- Def：物品、建筑、工作、需求、条件、思想、证据类型、异类、事件、派系、镇规。
- 模板：人物背景、关系、旅客组合、事件场景、证词文本。
- 规则组件：触发、感知、限制、痕迹、交换、传播、处置和后果。

## 数据流程

```text
JSON5 / CSV / localization
→ Schema 校验
→ 模组依赖与 Patch
→ 引用解析
→ 内容语义校验
→ 编译为不可变 Catalog
→ 内容 Hash
→ Simulation Worker
```

## 首发模组能力

允许：新增/修改 Def、贴图、音频、文本、规则图、事件、配方、镇规和本地化。

禁止：任意 JavaScript、Node API、动态网络请求、修改核心 Tick、直接访问组件存储。

## 内容缺失

存档加载时显示缺失 Def 与影响；可使用显式占位 Def 保留实体身份。关键内容缺失不得静默删除角色、债务或镇志证据。

## 官方内容质量

每个新增异类至少影响三套系统；每个事件必须拥有非文本后果；每条镇规必须有执行成本；每个派系必须能提供至少一种独特资源和一种长期约束。
