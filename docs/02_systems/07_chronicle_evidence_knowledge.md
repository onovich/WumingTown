# 镇志、证据与公共知识

## 对象模型

```text
CaseFile 案卷
├─ Observations 观察
├─ Testimonies 证词
├─ Traces 痕迹
├─ Sources 文献/来源
├─ Hypotheses 假设
├─ Contradictions 反证
├─ ConfirmedRules 已确认规则
└─ Dissemination 居民知晓范围
```

## 证据不是概率真值

每条证据记录来源、直接性、独立性、保存质量、感知条件、利益冲突和关联假设。系统可以计算内部支持度，但玩家界面主要显示“传闻/薄弱/可信/高度可信/确认”，避免伪精确。

确认规则至少要求：达到支持阈值、两类独立证据、无未解决致命矛盾。设计师可对特殊异类定义额外条件。

## 错误知识

假设可以错误；已写入镇志的规则也可以被新证据推翻。居民按照自己接触到的版本行动。政治势力可要求删改，但任何修改都留下版本历史和知情者。

## 调查动作

- 访谈与交叉询问
- 保存现场、尸检、物品鉴定
- 对照旧档案与外地记录
- 低风险控制实验
- 高风险诱发实验
- 与异类直接交涉

调查消耗时间、技能、材料、空间和伦理成本。

## 镇志损坏

火灾、潮湿、盗窃、政治审查、异类影响可损坏载体。知识可以存在于书、抄本和人物记忆中；冗余保存提高韧性。

## 自动行为

确认规则可编译为居民安全策略，例如“不回应第三次敲门”。未经确认的假设默认不自动执行，除非玩家主动制定临时镇规，并承担误判责任。

## 公平性规则

每个高危规则必须拥有可观察前兆；第一次致命遭遇前应有低风险线索或可退避路径；事故复盘列出可用证据、当前假设和错误推断链。
